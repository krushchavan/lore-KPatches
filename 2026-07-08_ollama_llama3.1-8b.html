[PATCH v3 1/2] perf record: Return the written size from process_comp_header() Neutral llm-chunked-no-replies Awaiting Review

This patch modifies the process_comp_header() function in perf record to return the number of bytes written or -1 if it won't fit, instead of just returning an increment. The caller is then responsible for checking whether a record fits within the remaining space. This change allows the function to reject records that are too large and prevents potential buffer overflows.

[PATCH v3 2/2] perf record: Fix multiple PERF_RECORD_COMPRESSED2 records per push Neutral llm-chunked-no-replies Awaiting Review

This patch fixes an issue where the perf record tool emits multiple PERF_RECORD_COMPRESSED2 records when a single mmap push exceeds the maximum record size due to Zstd compression. The problem occurs because the header.size field is truncated, causing write failures and mis-framed data. To fix this, the patch changes the process_comp_header function to set per-record data_size, 8-byte-align header.size, and zero trailing padding. This ensures that each record is written correctly, even when compressed output exceeds the maximum record size.

🔥 TODAY [PATCH v2] fs/proc/task_mmu: fix PAGEMAP_SCAN written state for PMD holes v1 → v2 Needs Work LLM (per-reviewer) Under Review

This patch fixes a bug in the PAGEMAP_SCAN functionality of the Linux kernel, specifically when dealing with PMD holes (large page table entries). The issue arises when an anonymous huge page is write-protected and then cleared using MADV_DONTNEED. As a result, the range is not reported as written, leading to stale data in subsequent incremental dumps. To fix this, the patch adds the PAGE_IS_WRITTEN category to the evaluation of holes in non-hugetlb uffd-wp VMAs, ensuring that PMD holes are correctly reported as written. A selftest has also been added to verify the fix.

[PATCH v2] mm/hugetlb: fix swap entry corruption when clearing uffd-wp at fork() Positive LLM Under Review

The patch fixes a swap entry corruption issue when clearing the uffd-wp bit at fork() in hugetlb pages. The problem arises from the different handling of the uffd-wp bit in migration and hwpoison entries, leading to incorrect clear operations. The patch corrects this by using pte_swp_clear_uffd_wp() for migration entries and removing the unnecessary clear operation on hwpoison entries.

Needs Work LLM (per-reviewer) Under Review

This patch series adds support to userfaultfd (a Linux kernel interface) for tracking the working set of virtual machine guest memory, allowing a Virtual Machine Manager (VMM) to identify hot pages and reclaim cold ones to slower storage. The approach taken is to introduce a new registration mode, UFFDIO_REGISTER_MODE_RWP, which uses the same mechanism as existing modes but with additional functionality for working set tracking. This enables the VMM to detect which pages are still being accessed, safely reclaim cold pages, and fetch them back on demand when needed. The patch series includes 15 patches that address various issues and conflicts, including rebasing onto mm-new, resolving a conflict with a hugetlb swap-corruption fix, and tolerating EINVAL from madvise(MADV_NOHUGEPAGE). The patches have been tested and show no regressions in the selftests/mm/uffd-unit-tests.
