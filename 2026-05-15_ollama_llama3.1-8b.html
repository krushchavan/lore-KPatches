Ongoing Submitted 2026-05-13 ⏱ No comments yet [PATCH 2/2] btrfs: test qgroup deletion races with squota writes Neutral llm-chunked-no-replies Awaiting Review

This patch adds four test cases to the Btrfs file system to cover various race conditions between qgroup deletion and squota writes. The tests aim to identify potential bugs in the handling of extent ownership references (EXTENT_OWNER_REF) when a subvolume is deleted or its qgroup is destroyed. The scenarios tested include freeing an extent whose owner qgroup is gone, creating an extent while the owning subvolume is being deleted, destroying a live subvolume's qgroup with delayed refs, and destroying a qgroup belonging to a still-mounted subvolume. The patch ensures that these operations are properly handled by the Btrfs kernel to prevent underflows or transaction aborts.

Ongoing Submitted 2026-05-05 🔥 TODAY [PATCH v6 0/7] locking: contended_release tracepoint instrumentation Needs Work LLM (per-reviewer) Under Review

This patch series adds a new tracepoint called contended_release that fires on the lock holder's side when a lock is released while there are waiters. This provides additional information such as hold time estimation and the holder's stack at release time, which can be used to measure how long a lock was held under contention. The patch also updates the behavior of reader/writer locks to fire the tracepoint for every reader releasing while a writer is waiting, not just the last reader.

Ongoing Submitted 2026-05-05 ⏱ No comments yet [PATCH v6 3/7] locking: Add contended_release tracepoint to sleepable locks Neutral llm-chunked-no-replies Awaiting Review

This patch adds a new tracepoint called contended_release to the Linux kernel's locking subsystem. The contended_release tracepoint fires when a lock is released under contention, allowing for correlation of lock hold time with waiter events by lock address. The patch modifies the slowpath unlock paths of various sleepable locks (mutex, rtmutex, semaphore, rwsem, percpu-rwsem, and RT-specific rwbase locks) to include calls to trace_contended_release(). This enables more detailed tracing and debugging of locking behavior in the kernel.

Ongoing Submitted 2026-05-05 🔥 TODAY [PATCH v6 5/7] locking: Add contended_release tracepoint to qspinlock Needs Work LLM (per-reviewer) Under Review

This patch adds a contended_release tracepoint to the qspinlock implementation, allowing for tracing of lock contention and release events. The patch uses an arch-overridable queued_spin_release() function to ensure correct behavior across all architectures, including those with custom unlock implementations. When the tracepoint is disabled, only a single NOP instruction is added to the hot path, minimizing performance impact. When enabled, the contention check, trace call, and unlock are combined in an out-of-line function to further minimize hot path impact. The patch also includes estimates of binary size increases due to this change.

Needs Work LLM (per-reviewer) Under Review

This patch series fixes several issues in the FUSE (Filesystem in Userspace) implementation for io-uring, a Linux I/O interface. The patches address races between fuse ring creation and abort, registration and abort, and moving cancelled entries to an in-userspace list. These issues can cause indefinite hangs during unmount and resource leaks. The approach taken involves modifying the FUSE code to ensure that request flushing always happens when cancelling operations.
