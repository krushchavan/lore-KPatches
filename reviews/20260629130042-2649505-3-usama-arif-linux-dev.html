Active on: 2026-06-29
Patch summary
This patch splits the v1 userspace eventfd code from mm/vmpressure.c into a new file, mm/vmpressure-v1.c. The split is done to clean up the original file and prepare for eventually making vmpressure CONFIG_MEMCG_V1 only. The patch moves almost half of the code in mm/vmpressure.c into the new file, which is built only when CONFIG_MEMCG_V1 is enabled. This results in a significant reduction in memory usage on systems where CONFIG_MEMCG_V1 is not enabled. The medium-term plan is to introduce a PSI-based socket-pressure path and eventually remove the tree=false path entirely.
