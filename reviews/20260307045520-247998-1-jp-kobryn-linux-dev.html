Patch summary
This patch adds the ability to track page allocations per memory policy, allowing for more detailed statistics and debugging capabilities. The problem it solves is the lack of visibility into how different memory policies are affecting system performance and resource allocation. The approach taken involves modifying the alloc_pages_mpol function to collect and report allocation statistics on a per-mempolicy basis.
JP (Meta) (author) 2026-03-13 Neutral LLM
Author acknowledged that existing numa_hit/miss/foreign counters don't account for node masks, but didn't commit to changing them or adding new counters; instead proposed reducing the number of new counters in the patch and moving them off of memcg. Author also clarified that direct node allocations are included in these stats.

applications.  However, IMHO, the equivalent memory policy is
