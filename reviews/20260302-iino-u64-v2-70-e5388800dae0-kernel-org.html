[PATCH v2 070/110] affs: replace PRIino with %llu/%llx format strings

View on lore.kernel.org →
Active on: 2026-03-02
Patch summary
This patch replaces the PRIino format macro with concrete format strings %llu and %llx in the AFFS (Amiga File System) code, which is now using u64 for inode numbers.
Jeff Layton (author) 2026-03-02 Neutral LLM
The author is responding to a review comment about the patch replacing PRIino format macro with concrete format strings in affs. The reviewer had pointed out that some uses of %llu and %llx were incorrect, but the author simply re-posted the original patch description without addressing the specific issues.
Show original comment 
Now that i_ino is u64 and the PRIino format macro has been removed,
replace all uses in affs with the concrete format strings.

Signed-off-by: Jeff Layton <jlayton@kernel.org>
