Patch summary
This patch modifies the Linux kernel's compaction code to support non-movable direct compaction for pageblock requests. It allows whole-block requests, which are essentially type neutral, to scan blocks of other types and fixes the protection gates to take the requested order into account. The only exception is CMA (Contiguous Memory Allocation) blocks, which remain strictly typed due to their sticky nature. This change aims to improve the efficiency of non-movable direct compaction, particularly in defrag_mode where whole-block production is a goal.
Johannes Weiner (author) 2026-07-01 Neutral LLM
The author is addressing Vlastimil's concern about non-movable compaction requests being allowed to scan movable blocks, even though they are not compactable anymore. The author acknowledges this issue and explains that the behavior for movable and non-movable requests is asymmetrical, with movable requests first trying high-effort mode on unmovable blocks before escalating, whereas non-movable requests start in high-effort mode but can escalate when it doesn't produce results.
