Patch summary
This patch converts the iomap operations for XFS to use the new ->iomap_next() callback, which is a more efficient and flexible way of handling I/O mappings. The conversion uses the iomap_process() helper function to finish previous mappings and produce the next one. This change does not introduce any functional changes but rather updates the internal implementation of XFS's iomap operations. The patch affects several files in the fs/xfs directory, including xfs_file.c and xfs_iomap.c.
