Patch summary
This patch fixes an issue where dirty folios are throttled too aggressively in cgroup v1 for MGLRU (Mostly Gone, Least Recently Used). The problem arises from the fact that the throttling check happens after a full scan of the given memory cgroup's LRU vector, leading to more aggressive throttling than traditional LRU. The patch aims to address this issue by adjusting the throttling behavior.
