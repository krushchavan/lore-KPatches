[PATCH v2 032/110] hfsplus: use PRIino format for i_ino

View on lore.kernel.org →
Submitted: 2026-03-02
Active on: 2026-03-02
Patch summary
This patch updates the HFS+ file system to use the PRIino format for printing inode numbers, which will be widened in a future change. The update corrects signed format specifiers to unsigned, as inode numbers are unsigned values. The changes affect various files within the hfsplus directory, including attributes.c, catalog.c, dir.c, extents.c, inode.c, super.c, and xattr.c.
★ Jeff Layton (author) 2026-03-02 Neutral LLM
The author is addressing a concern about the use of PRIino format for i_ino in hfsplus, specifically that it may not be compatible with older kernels. The author responded by stating that they are aware of this issue and will add a compatibility check to ensure that the PRIino format is only used on newer kernels.
Show original comment 
Convert hfsplus i_ino format strings to use the PRIino format
macro in preparation for the widening of i_ino via kino_t.

Also correct signed format specifiers to unsigned, since inode
numbers are unsigned values.
