Patch summary
This patch adds the ability for drivers using dax_kmem to specify an auto-online policy when creating a DAX device. It introduces a new field, 'online_type', which allows drivers to override the system default policy. The system default policy is now resolved at kmem driver bind time, rather than at device creation time, to avoid freezing stale policies. This change does not affect existing callers, as they still use the system default policy.
