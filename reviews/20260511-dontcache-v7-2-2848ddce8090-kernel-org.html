[PATCH v7 2/3] mm: track DONTCACHE dirty pages per bdi_writeback

View on lore.kernel.org →
Submitted: 2026-05-11
Active on: 2026-05-11
Patch summary
This patch adds a new counter, WB_DONTCACHE_DIRTY, to track the number of dirty pages that have been dirtied via RWF_DONTCACHE writes. The counter is incremented when a folio is dirtied and decremented when it is cleaned or cleared. This information will be used by the writeback flusher to determine how many pages to write back when expediting writeback for IOCB_DONTCACHE writes, without flushing the entire BDI's dirty pages. The patch also updates the inode_do_switch_wbs function to transfer the counter alongside WB_RECLAIMABLE when an inode switches cgroup writeback domains.
