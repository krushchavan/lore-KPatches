[PATCH v5 2/4] firmware: arm_sdei: add SDEI_EVENT_SIGNAL support

View on lore.kernel.org →
Submitted: 2026-06-30
Active on: 2026-06-30
Patch summary
This patch adds a new function, sdei_event_signal(), which is a thin wrapper over the SDEI_EVENT_SIGNAL call to signal software-signalled events (event 0) to another PE in an NMI-like manner. The function takes no locks and is safe to call from NMI or crash context.
