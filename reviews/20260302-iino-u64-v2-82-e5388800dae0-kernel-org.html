[PATCH v2 082/110] ext2: replace PRIino with %llu/%llx format strings

View on lore.kernel.org →
Active on: 2026-03-02
Patch summary
This patch updates the ext2 file system to use explicit format strings instead of the deprecated PRIino macro, which is no longer needed since inode numbers are now u64 type. The changes affect several files in the ext2 directory and correct formatting issues in error messages.
Jeff Linux-f2fs-devel 2026-03-02 Reviewed-by Neutral LLM
The reviewer noted that the patch does not address all instances of PRIino in ext2 code, specifically mentioning ext2_xattr.c where PRIino is still used.
Show original comment 
Now that i_ino is u64 and the PRIino format macro has been removed,
replace all uses in ext2 with the concrete format strings.
