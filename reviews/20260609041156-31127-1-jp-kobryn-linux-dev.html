Patch summary
This patch adds two new tracepoints to the Linux kernel's memory management (mm) subsystem: mm_lru_add_drain and mm_lru_drain_all_queue. The first tracepoint reports when an LRU add batch is drained, allowing tracing of partial drains and attribution to calling stacks. The second tracepoint captures when per-CPU drain work is queued for remote drain operations, including the requester stack and target CPU. These additions improve debugging capabilities by providing more detailed information about memory management events.
