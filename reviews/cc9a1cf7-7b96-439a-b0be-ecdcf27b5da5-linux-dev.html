Active on: 2026-05-29
Patch summary
This patch modifies the force_thp_readahead order calculation to use mapping_max_folio_order() instead of a hardcoded value. The previous code had issues with memory pressure due to large file folios, and this change aims to provide a controlled way to allocate larger folios on architectures that currently don't support them. The new approach sets the thp_order to the minimum of the maximum folio order for the mapping and 2M, ensuring that the allocation is not too aggressive and causing excessive memory pressure. This patch also removes an unnecessary min3() call in the original code.
