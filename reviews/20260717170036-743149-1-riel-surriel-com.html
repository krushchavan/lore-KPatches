[PATCH v3 0/6] mm: access remote process memory under the per-VMA lock

View on lore.kernel.org →
Submitted: 2026-07-20
Active on: 2026-07-20
Patch summary
This patch series improves the way Linux accesses remote process memory by introducing a per-VMA lock in __access_remote_vm(). The per-VMA lock is used when accessing memory within a single VMA, and falls back to the mmap lock for cross-VMA accesses or when other conditions are met. A new gup helper, get_user_page_vma(), is also introduced to fault in missing pages under either lock. This change aims to reduce contention on the mmap lock and improve system reliability by preventing system monitoring software from getting stuck behind a locked process.
