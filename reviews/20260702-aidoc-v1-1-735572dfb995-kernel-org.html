Patch summary
This patch removes the requirement for attributing Large Language Model (LLM) assistance in kernel documentation, aiming to clarify the distinction between meaningful and trivial LLM contributions.
Laurent Pinchart 2026-07-02 Needs Work LLM
Laurent Pinchart noted that even if the Assisted-by tag is removed, it is still necessary to mention the requirement for disclosing usage of code-generation tools in Documentation/process/generated-content.rst.

roughly:

in the commit message:

patch submitters to note which part(s) of the patch had machine
assistance applied.  I don't care all that much about the technology
used (e.g. LLM, or Eliza, or whatever).  Something like:

Assisted-by: LLM # commit message
Assisted-by: LLM # finding bugs in the original commit
