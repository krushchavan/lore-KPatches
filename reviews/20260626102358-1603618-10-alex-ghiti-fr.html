Patch summary
This patch updates zswap and zsmalloc to use per-node object cgroup (obj_cgroup) for kernel memory (kmem) accounting, attributing compressed page charges to the correct NUMA node. However, it does not correctly handle entries that straddle pages across multiple nodes, which will be addressed in a separate series by Joshua Hahn. The patch adds a function `zs_handle_to_nid` to determine the NUMA node for a given zsmalloc handle and updates the `zswap_store_page` function to use this information. This change is necessary to ensure accurate kmem accounting for compressed pages.

update or move the object behind the handle. This helper does the same decode
without the lock, so zswap's uncharge path can race migration and charge or
uncharge the wrong node, or observe transient zspage state.
