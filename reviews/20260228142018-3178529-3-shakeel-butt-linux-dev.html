[PATCH 2/3] cgroup: add lockless fast-path checks to cgroup_file_notify()

View on lore.kernel.org →
Submitted: 2026-02-28  ·  Last comment: 2026-03-05
Active on: 2026-02-28
Patch summary
Add two lockless checks before acquiring the lock: 1. READ_ONCE(cfile->kn) NULL check to skip torn-down files. 2. READ_ONCE(cfile->notified_at) check to skip when within the rate-limit window (~10ms). Both checks have safe error directions -- a stale read can only cause unnecessary lock acquisition, never a missed notification. Annotate all write sites with WRITE_ONCE() to pair with the lockless readers. The trade-off is that trailing timer_reduce() calls during bursts are skipped, so the deferred notification that delivers the final state may be lost. This is acceptable for the primary callers like __memcg_memory_event() where events keep arriving.
Chen Ridong 2026-02-28 Inline Review Neutral Heuristic
Good series to move away from the global lock. Previously, if a notification arrived within the rate-limit window, we would still call timer_reduce(&cfile->notify_timer, next) to schedule a deferred notification. With this change, returning early here bypasses that timer scheduling entirely. Does this risk missing notifications that would have been delivered by the timer?
