Patch summary
This patch extends the passthrough functionality in the fuse file system to support inode operations, allowing for improved performance and reduced latency in certain use cases. The main goal is to eliminate round trips for getattr and open operations, which can be a bottleneck for short-lived services and batch jobs that need to fetch large data over the network. The patch aims to add a lazy materialization layer using fuse that supports on-demand fetching of cold data, switching to passthrough mode once the required content has been fetched locally. This will provide near-native performance with minimal userspace overhead. The initial implementation focuses on read-only operations and includes support for open + read passthrough, getattr/statx passthrough, and readdirplus. Future work may include additional features such as handling cross-boundary moves and other edge cases.
