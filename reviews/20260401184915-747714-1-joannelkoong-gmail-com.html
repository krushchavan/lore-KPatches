Patch summary
This patch fixes a stall in the io-uring background queue when a request completes. The issue arises from the fact that the connection-level background counters are not properly accounted for before flushing the queue's background queue, leaving pending requests stuck. To fix this, the patch accounts for the connection-level counters first and then flushes the queue's background queue, ensuring that pending requests are dispatched correctly.

Bernd Schubert 2026-04-01 Reviewed-by Needs Work LLM
The reviewer, Bernd Schubert, disagrees that the patch fixes a real issue and suggests an alternative solution to prevent background requests from getting stuck in the io-uring per-queue bg queue. He points out that fuse_uring_flush_bg() already allows at least one background request per queue to pass through when fc->active_background is below the limit, and proposes to always let through at least one request per queue. The reviewer noted that the patch fixes an issue where only one request can run at a time, but did not provide further technical details or suggestions for improvement. The reviewer noted that there might be a stuck background queue issue at teardown, and mentioned that Horst is investigating this separately.
