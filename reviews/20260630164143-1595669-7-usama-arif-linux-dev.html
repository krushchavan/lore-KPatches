[PATCH 6/6] mm: rename ARCH_ENABLE_THP_MIGRATION to ARCH_SUPPORTS_PMD_SOFTLEAF

Active on: 2026-07-01 • 2026-06-30
Patch summary
This patch renames the CONFIG_ARCH_ENABLE_THP_MIGRATION configuration option to CONFIG_ARCH_SUPPORTS_PMD_SOFTLEAF, which more accurately reflects its purpose of enabling PMD-level softleaf machinery. The rename is a pure change and does not introduce any functional changes. The affected architectures remain unchanged, with x86, arm64, s390, riscv, loongarch, and powerpc on PPC_BOOK3S_64 still selecting this option. This change aims to improve the clarity of the configuration options.
David (Arm) 2026-07-01 Reviewed-by Neutral LLM
Reviewer David pointed out that the description of CONFIG_ARCH_ENABLE_THP_MIGRATION as having 'grown to gate the entire PMD-level softleaf machinery' is inaccurate, specifically mentioning device-private entries which rely on migration support and are not affected by the rename. Reviewer David (Arm) suggested using either ARCH_HAS_PMD_SOFTLEAVES or HAVE_ARCH_PMD_SOFTLEAVES instead of ARCH_SUPPORTS_PMD_SOFTLEAF, and agreed that the config option rename makes sense.
