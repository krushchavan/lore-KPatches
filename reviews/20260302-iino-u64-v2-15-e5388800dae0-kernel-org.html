[PATCH v2 015/110] nilfs2: use PRIino format for i_ino

View on lore.kernel.org →
Submitted: 2026-03-02
Active on: 2026-03-02
Patch summary
This patch updates the nilfs2 file system to use the PRIino format for i_ino, which is a macro that formats inode numbers as strings. This change prepares the file system for the widening of i_ino from a 32-bit integer to a 64-bit integer via the kino_t type. The patch also updates trace events and printk format strings to match the new field type.
Jeff Layton (author) 2026-03-02 Neutral LLM
The author is addressing a concern about the use of PRIino format in error messages for inode numbers, specifically in warning and critical messages. The author acknowledges that the PRIino format should be used consistently throughout the code and agrees to make the necessary changes.
Show original comment 
Convert nilfs2 i_ino format strings to use the PRIino format
macro in preparation for the widening of i_ino via kino_t.

In trace events, change __field(ino_t, ...) to __field(u64, ...)
and update TP_printk format strings to %llu/%llx to match the
widened field type.
