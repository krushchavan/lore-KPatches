Patch summary
This patch adds a new ioctl (input/output control) called BTRFS_IOC_GET_CSUMS, which allows userspace to query the on-disk checksums for a file without needing to read the data itself. The ioctl takes an argument specifying the offset and length of interest, as well as a buffer to store the results. The kernel returns a struct containing information about whether the checksum is available, and if so, what type it is (e.g., sparse, preallocated, or compressed). This feature is primarily intended for speeding up mkfs.btrfs --rootdir operations, particularly when using the --reflink option, which can result in significant performance improvements.
Qu Wenruo 2026-04-04 Inline Review Reviewed-by Needs Work LLM
Qu Wenruo expressed concern that allowing non-root users to use the BTRFS_IOC_GET_CSUMS ioctl could lead to information leakage due to potential bad inputs, but noted that even if a user has access to fiemap results for files utilizing certain byte numbers, they would still require proper privilege to call fiemap on those files. Qu Wenruo is open to requiring root privilege for the ioctl as an alternative. Reviewer Qu Wenruo noted that the current implementation of BTRFS_IOC_GET_CSUMS ioctl simply returns whatever range and csum values are requested by userspace, without any validation or checks for potential issues such as bookend or compressed extents. He suggested that this approach is a variant of TREE_SEARCH, but not suitable for csum tree search due to its lack of robustness.
