[PATCH v2 017/110] affs: use PRIino format for i_ino

View on lore.kernel.org →
Submitted: 2026-03-02
Active on: 2026-03-02
Patch summary
This patch updates the affs file system to use the PRIino format for printing inode numbers, which will be widened in a future change via the kino_t type. The update corrects signed format specifiers to unsigned, as inode numbers are unsigned values. This change affects various files within the affs codebase, including amigaffs.c, bitmap.c, dir.c, file.c, inode.c, namei.c, and symlink.c.
Jeff Layton (author) 2026-03-02 Neutral LLM
The author is addressing a concern about the use of PRIino format in debug messages for inode numbers, which are unsigned values. The author acknowledges that the patch needs to correct signed format specifiers and agrees to restructure the code in v2.
Show original comment 
Convert affs i_ino format strings to use the PRIino format
macro in preparation for the widening of i_ino via kino_t.

Also correct signed format specifiers to unsigned, since inode
numbers are unsigned values.
