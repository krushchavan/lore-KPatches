[PATCH v2 0/7] mm: switch THP shrinker to list_lru

View on lore.kernel.org →

Active on: 2026-03-18
Patch summary
This patch series switches the THP (Transparent Huge Pages) shrinker to use the list_lru data structure instead of its current implementation. The motivation for this change is to address scalability issues in the original implementation, which involved a complex sequence of function calls. The new approach uses the list_lru API to manage the LRU state and allows for more fine-grained control over locking. The patches also include various cleanups and refactors to make the code easier to review and maintain.
★ Shakeel Butt 2026-03-18 Acked-by Neutral LLM
Shakeel Butt suggested squashing his cleanup patch into this one, citing that the lock_list_lru() function has a common pattern for bailing in both MEMCG and !MEMCG variants, which can be consolidated into a single implementation.
