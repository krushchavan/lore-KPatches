[PATCH v2 018/110] afs: use PRIino format for i_ino

View on lore.kernel.org →
Submitted: 2026-03-02
Active on: 2026-03-02
Patch summary
This patch updates the AFS (Andrew File System) code to use the PRIino format for printing inode numbers, in preparation for the widening of inode types via kino_t. The changes affect several files within the fs/afs directory and involve replacing string formatting macros with the PRIino macro. This change is necessary to accommodate future inode type expansions.
★ Jeff Layton (author) 2026-03-02 Neutral LLM
Author addressed a concern about the PRIino format not being used consistently in debug messages, explaining that it is only necessary to use PRIino for inode numbers when they are printed as part of a larger string and provided an example where it was unnecessary.
