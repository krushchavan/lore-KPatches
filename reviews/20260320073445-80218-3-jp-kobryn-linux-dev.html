[PATCH 2/2] btrfs: prevent direct reclaim during compressed readahead

View on lore.kernel.org →
Submitted: 2026-03-20
Active on: 2026-03-20
Patch summary
This patch prevents direct reclaim during compressed readahead in Btrfs by passing specific GFP flags when allocating memory for readahead operations. The new gfp_t API is used to restrict direct reclaim, and a bool parameter is added to the add_ra_bio_pages function to allow callers to specify whether direct reclaim should be restricted or not. This change affects the btrfs_submit_compressed_read function and the add_ra_bio_pages function in fs/btrfs/compression.c.
