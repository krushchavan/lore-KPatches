Patch summary
This patch series addresses a memory regression issue caused by not reclaiming unused portions of 2M THP folios on ARM, which can occur when deploying 2M THP=always. The patches modify the thp_underused() function to work with non-PMD-sized folios and add anonymous mTHP folios to the deferred split list from the anonymous fault/collapse path, allowing the shrinker to find and split them. This approach is compatible with existing PMD THP collapse thresholds and ensures that unused portions of 2M THPs can be broken down and reclaimed, preventing memory regressions.

"Does this scale calculation unintentionally disable underused splitting for

Due to integer division truncation, max_ptes_none >= nr_pages - 1 can match

...
Should the sentinel check remain against the raw
khugepaged_max_ptes_none value before scaling?"

Sashiko is right. I'll fix this up in v2 and keep the original

non-scaled value.
