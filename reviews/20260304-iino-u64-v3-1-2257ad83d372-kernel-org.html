[PATCH v3 01/12] vfs: widen inode hash/lookup functions to u64

View on lore.kernel.org →
Submitted: 2026-03-04  ·  Last comment: 2026-03-06
Active on: 2026-03-04
Patch summary
Change the inode hash/lookup VFS API functions to accept u64 parameters instead of unsigned long for inode numbers and hash values. This is preparation for widening i_ino itself to u64, which will allow filesystems to store full 64-bit inode numbers on 32-bit architectures. Since unsigned long implicitly widens to u64 on all architectures, this change is backward-compatible with all existing callers.
