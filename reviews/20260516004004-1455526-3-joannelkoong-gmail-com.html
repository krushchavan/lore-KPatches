[PATCH v2 02/21] fuse: prepare for passthrough of inode operations

View on lore.kernel.org →
Submitted: 2026-05-18
Active on: 2026-05-18
Patch summary
This patch prepares the fuse filesystem to pass through inode operations by adding a new field 'ops_mask' to the FUSE_DEV_IOC_BACKING_OPEN ioctl input argument. The ops_mask declares which inode operations will be passed through to the backing file once it's attached to the fuse inode on lookup. Currently, only regular files are allowed for read/write/splice/mmap passthrough, but this patch relaxes that restriction by allowing non-regular backing files when setting operations other than FUSE_READ/FUSE_WRITE in ops_mask. Additionally, a filesystem with FUSE_PASSTHROUGH_INO is required to set up passthrough of any inode operations for simplification of inode attribute caching.
