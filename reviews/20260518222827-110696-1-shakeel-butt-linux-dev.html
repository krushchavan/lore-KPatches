[PATCH v3] memcg: cache obj_stock by memcg, not by objcg pointer

Patch summary
This patch fixes a performance regression in the Linux kernel's memory control group (memcg) implementation, specifically when dealing with cross-NUMA workloads. The issue arises from the fact that obj_stock_pcp caches are keyed by pointer to obj_cgroup instead of memcg, leading to unnecessary drains and misses on every refill and consume operation. To fix this, the patch changes the cache keying to use memcg instead of obj_cgroup pointer, allowing siblings to share the reserve without re-caching. This temporary fix will be reverted when per-node kmem accounting is introduced.
