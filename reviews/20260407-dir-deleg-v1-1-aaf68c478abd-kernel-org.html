[PATCH 01/24] filelock: add support for ignoring deleg breaks for dir change events

Patch summary
This patch adds support to ignore delegation breaks for directory change events in the Linux kernel's filelock module. The patch introduces a new flags parameter to try_break_deleg() and modifies __break_lease to ignore certain types of delegation break events, specifically those related to directory changes such as creation, deletion, or renaming. This allows NFS clients to be notified of directory changes after the fact instead of having the server recall the delegation immediately. The patch also includes modifications to several other files in the kernel's file system code to accommodate this new behavior.

★ Chuck Lever 2026-04-07 Inline Review Reviewed-by Neutral LLM
The reviewer noted that after breaking visible leases, the code calls any_leases_conflict() which doesn't filter out ignored dir-delegation leases, causing a loop when only ignored leases remain.
Show original comment 
After breaking visible leases, the restart: label calls any_leases_conflict()
which does not filter ignored dir-delegation leases. When only ignored leases
remain, any_leases_conflict returns true, but visible_leases_remaining also
returned true (triggering the wait). The code picks the first lease (possibly
ignored), computes break_time = 1 jiffy, blocks, then loops.                                                     

For example, suppose you have two directory delegations on a directory, one
with FL_IGN_DIR_DELETE and one without. After the non-ignored one is broken
and removed, the ignored one keeps any_leases_conflict returning true. The
loop spins at 1-jiffy intervals until the ignored delegation is released.  

Should the restart: block skip ignored leases?
