[PATCH v4 4/5] mm: rename zone->lock to zone->_lock

View on lore.kernel.org →
Active on: 2026-03-04 • 2026-03-03 • 2026-03-02 • 2026-02-28 • 2026-02-27
Patch summary
This patch renames the lock field in Linux kernel memory zones from zone->lock to zone->_lock, but updates documentation to consistently refer to the lock using the wrapper API (zone_lock_* and zone_unlock_*) rather than directly accessing the internal _lock field.
