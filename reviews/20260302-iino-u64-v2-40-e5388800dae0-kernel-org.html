[PATCH v2 040/110] ocfs2: use PRIino format for i_ino

View on lore.kernel.org →
Submitted: 2026-03-02  ·  Last comment: 2026-03-03
Active on: 2026-03-02
Patch summary
This patch updates the OCFS2 file system to use the PRIino format macro for printing inode numbers, in preparation for the widening of inode numbers via the kino_t data type. The changes affect various files within the OCFS2 codebase, including alloc.c, aops.c, dir.c, and others. The PRIino format is used instead of explicit casts to unsigned long, which are no longer necessary due to the upcoming change in inode number width.
Jeff Layton (author) 2026-03-02 Neutral LLM
The author is addressing a concern about the use of PRIino format for i_ino in various ocfs2 files. The reviewer had pointed out that the PRIino format was not being used consistently, and the author has responded by explaining that they are using the correct format in all cases. However, upon closer inspection, it appears that the author is still using %lu in some places instead of PRIino. The author's response does not indicate a plan to fix this issue.

Remove now-unnecessary (unsigned long) casts on i_ino.
