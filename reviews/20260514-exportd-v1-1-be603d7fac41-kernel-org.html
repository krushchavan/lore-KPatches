[PATCH] exportfs: drop unused is_export parameter from xtab_read() and xtab_write()

View on lore.kernel.org →
Submitted: 2026-05-14
Active on: 2026-05-14
Patch summary
This patch removes the unused 'is_export' parameter from two functions, xtab_read() and xtab_write(), in the exportfs code. The parameter is always passed as 1, so it's unnecessary and can be simplified by eliminating dead code paths.
