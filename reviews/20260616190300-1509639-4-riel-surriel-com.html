[PATCH 3/3] mm: read remote memory without the mmap lock where possible

View on lore.kernel.org →
Submitted: 2026-06-19
Active on: 2026-06-19
Patch summary
This patch adds an opportunistic, read-only fast path to the Linux kernel's memory access functions that allows reading remote memory without taking the mmap lock when possible. The existing slow path still uses the mmap lock for reads and writes. The new fast path checks VMA permissions and grabs a short-lived reference to a present page before copying it out. If the page is not present, or if there are other issues like hugepages or I/O mappings, it falls back to the slow path. This change aims to reduce contention on large machines by avoiding unnecessary locking for common read operations.
