[PATCH v2 065/110] nfs: replace PRIino with %llu/%llx format strings

View on lore.kernel.org →
Active on: 2026-03-02
Patch summary
This patch replaces the use of the PRIino format macro in the NFS code with explicit %llu and %llx format strings, reflecting that inode numbers are now u64 values.
Jeff Layton (author) 2026-03-02 Neutral LLM
The author is addressing a concern about the format strings used in nfs code, specifically replacing PRIino with %llu/%llx to match the new type of i_ino as u64. The author's response is neutral, providing context and explanation for their patch without committing to any changes or acknowledging a need for further revision.
Show original comment 
Now that i_ino is u64 and the PRIino format macro has been removed,
replace all uses in nfs with the concrete format strings.

Signed-off-by: Jeff Layton <jlayton@kernel.org>
