[PATCH 1/2] btrfs: don't finalize fs roots during qgroup snapshot accounting

Patch summary
This patch modifies the Btrfs file system to avoid finalizing file system roots during qgroup snapshot accounting, which can lead to incorrect state and silent corruption of snapshots. The patch introduces a new helper function `prepare_root_item` that updates root items without the side effects of finalization, and a new function `qgroup_commit_fs_roots` that iterates over dirty roots using an advancing index instead of clearing tags. This change prevents premature clearing of `BTRFS_ROOT_FORCE_COW`, freeing of log trees, and other finalization actions during transaction progress, ensuring consistent qgroup accounting and preventing potential corruption issues.
Sun YangKai 2026-04-24 Inline Review Reviewed-by Needs Work LLM
Reviewer Sun YangKai pointed out that the patch is generally well-received but requested minor implementation details to be addressed, specifically suggesting improvements to the code's organization and naming conventions. Reviewer Sun YangKai suggested using an unsigned int for the 'ret' variable, as it's only set by radix_tree_gang_lookup_tag which returns an unsigned int, and also questioned the name 'ret' since it's never returned. Reviewer Sun YangKai suggested modifying the index calculation in qgroup_commit_fs_roots() to prevent potential array out-of-bounds access, proposing a change from `index = ret;` to `index=btrfs_root_id(gang[ret-1]) + 1;`. Reviewer suggested optimizing the code by updating the 'index' variable outside the loop, which is not used inside the loop, to improve readability.
