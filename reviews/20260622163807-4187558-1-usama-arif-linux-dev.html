Patch summary
This patch updates the smp code to use release stores for publishing per-CPU CSD debug state, ensuring that preceding stores are ordered before a remote reader acquires the pointer. The change replaces open-coded barriers and uses smp_store_release() instead of plain stores, which matches the load-acquire operation used by csd_lock_wait_toolong(). This improves ordering and reduces unnecessary barriers in the clear path.
Usama Arif (author) 2026-06-24 Neutral LLM
The author addressed Kunwu's concern that the patch does not preserve the original ordering of diagnostic marker updates and callback/unlock work. The author explained that the patch still ensures the correct ordering between CSD execution/reuse and the clear path, but relaxes the historical implementation around the diagnostic marker.
