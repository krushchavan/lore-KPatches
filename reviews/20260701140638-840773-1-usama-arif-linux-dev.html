[PATCH] mm/migrate_device: pin large folios before splitting

Patch summary
This patch fixes a bug in the device migration code where large folios were not properly pinned before being split. The issue occurred when migrate_vma_collect_pmd() detected a large folio while holding the PTE lock, then dropped the lock before calling migrate_vma_split_folio(). This allowed concurrent reclaim or truncation to replace or clear the entry and drop the last reference to the folio. To fix this, the patch takes a temporary reference on the folio before dropping the PTE lock and passes that reference into migrate_vma_split_folio(), ensuring that the split helper sees only the expected caller pin and preventing potential failures.
