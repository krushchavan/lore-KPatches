[PATCH v2 046/110] udf: use PRIino format for i_ino

View on lore.kernel.org →
Active on: 2026-03-02
Patch summary
This patch updates the UDF file system to use the PRIino format for printing inode numbers, in preparation for the widening of i_ino via kino_t. The changes are made across five files: directory.c, file.c, inode.c, namei.c, and super.c. The PRIino format is used instead of the previous %lu format to ensure that inode numbers are printed correctly even when they exceed 32 bits. This change will help prevent issues with large inode numbers in the UDF file system.
Jeff Linux-f2fs-devel 2026-03-02 Neutral LLM
The reviewer noted that the patch does not address all instances of the format string in the code, specifically mentioning that the changes are incomplete as there are still some occurrences of %lu in the code.
