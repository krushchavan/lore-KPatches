[PATCH v2 016/110] 9p: use PRIino format for i_ino

View on lore.kernel.org →
Active on: 2026-03-02
Patch summary
This patch updates the 9p file system to use the PRIino format macro for printing inode numbers (i_ino). This is a preparatory step for widening i_ino via kino_t, which will allow it to hold larger values. The changes affect several files in the fs/9p directory and involve replacing hardcoded format strings with the PRIino macro.
Jeff Layton (author) 2026-03-02 Neutral LLM
The author is addressing a concern about the PRIino format being used in debug messages, specifically that it should be used consistently throughout the code. The author agrees to use PRIino consistently and does not mention any plans for further revision.
