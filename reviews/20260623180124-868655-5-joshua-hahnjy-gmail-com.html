Patch summary
This patch converts the memory control (memcg) system to use page_counter_stock instead of its own stock handling logic. This change removes special casing for spinning and refilling, allows for failure during allocation if there's not enough memory, and updates drain_all_stock and memcg_hotplug_cpu_dead to use the new stock drain variant. As a result, memsw and memory page counters now have independent stocks, which can lead to transient discrepancies in reported usage.
