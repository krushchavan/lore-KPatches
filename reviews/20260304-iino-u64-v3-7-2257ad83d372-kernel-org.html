[PATCH v3 07/12] hugetlbfs: widen trace event i_ino fields to u64

View on lore.kernel.org →
Submitted: 2026-03-04
Active on: 2026-03-04
Patch summary
This patch updates the hugetlbfs trace event definitions to use u64 instead of ino_t/unsigned long for inode number fields, allowing for wider range and more precise tracking of inode numbers.
Jeff Linux-f2fs-devel 2026-03-04 Inline Review Neutral LLM
reviewer noted that the u64 fields in hugetlbfs trace events are not being printed correctly due to a missing format specifier, and requested the addition of %llu for printing 64-bit unsigned longs
