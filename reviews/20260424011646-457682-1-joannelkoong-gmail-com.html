Patch summary
This patchset is a series of cleanups in the fuse: dir.c file, addressing code structure issues pointed out by Amir. The changes aim to improve the maintainability and readability of the code without introducing any functional changes, except for the first patch which fixes missing forget requests in error paths.
Joanne Koong (author) 2026-04-23 Positive LLM
The author addressed a concern about missing forget requests in error paths, explaining that these paths only trigger with misbehaving servers returning invalid attributes and that the real-world impact is minimal. The author agreed to send a forget request to tell the server to release its reference.
Show original comment 
If the server returned a valid nodeid but invalid attributes, the kernel
needs to send a forget request to tell the server to release its
reference, else this leads to server-side leaks. These paths only
trigger with a misbehaving server returning invalid attributes, so the
real-world impact is minimal, but forgets should be sent.
