Patch summary
This patch adds a new unprivileged ioctl command called BTRFS_IOC_GET_CSUMS, which allows userspace to query the on-disk checksums for a file without requiring root privileges. The ioctl takes an offset and length as input, and returns a struct btrfs_ioctl_get_csums_entry containing information about the checksums. This is particularly useful for speeding up mkfs.btrfs operations, especially when using the --reflink option, which can result in significant performance improvements.

Reviewer noted that mkfs will skip the range as a hole, making it even faster than using precalculated csums, and suggested keeping the ZEROED flag for future users Reviewer suggested adding an ENCODED flag to indicate if a range is compressed or encrypted, to maintain consistency with existing ZEROED flag for prealloc/hole extents. Reviewer noted that the patch does not handle the case where the requested offset is beyond the EOF, and suggested adding a check to return an error in this case. The reviewer suggested adding more explanation about the output buffer format and proposed modifying btrfs_ioctl_get_csums_args to include a flag/version member for future changes or additions.
