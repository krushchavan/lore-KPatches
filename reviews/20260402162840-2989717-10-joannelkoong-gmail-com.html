[PATCH v2 09/14] fuse: rearrange io-uring iovec and ent allocation logic

View on lore.kernel.org →
Submitted: 2026-04-02  ·  Last comment: 2026-04-15
Active on: 2026-04-02
Patch summary
This patch rearranges the io-uring iovec and ent allocation logic in fuse, moving the get_iovec_from_sqe function to an earlier location and reordering the allocation logic within fuse_uring_create_ring_ent. The change is intended to make it easier for future commits that add buffer rings.
Bernd Schubert 2026-04-02 Reviewed-by Neutral LLM
Expressed neutral sentiment about the change, stating that it doesn't align with their original goal of doing checks before allocation, but is willing to accept it if useful for future commits.
