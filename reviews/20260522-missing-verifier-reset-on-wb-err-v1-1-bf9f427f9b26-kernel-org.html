[PATCH] nfsd: reset write verifier on deferred writeback errors

View on lore.kernel.org →
Submitted: 2026-05-22
Active on: 2026-05-22
Patch summary
This patch fixes a bug in the NFS server where deferred writeback errors were not properly handled, leading to silent data loss. The issue arises when clients hold UNSTABLE write data under the current verifier and then COMMIT, only to receive the unchanged verifier back and conclude their data is durable. To fix this, the patch adds calls to commit_reset_write_verifier() at two error sites in nfsd_vfs_write() and nfsd_commit(), ensuring that the server's write verifier is properly rotated on deferred writeback errors. This change aligns with existing patterns in the code and ensures compliance with the UNSTABLE+COMMIT durability contract.
Chuck Lever 2026-05-22 Reviewed-by Needs Work LLM
Reviewer noted that the patch does not handle the case where filemap_check_wb_err() returns an error due to a writeback failure, and suggested adding a call to commit_reset_write_verifier() in this scenario. Reviewer noted that the patch does not handle the case where filemap_check_wb_err() returns -EAGAIN or -ESTALE, and requested adding a check to filter these errors before calling commit_reset_write_verifier().
