[PATCH v1 00/17] fuse: extend passthrough to inode operations

View on lore.kernel.org →
Submitted: 2026-04-22
Active on: 2026-04-22
Patch summary
This patch series extends the FUSE passthrough feature to support inode operations, including getattr and setattr, directory readdir, and kernel-initiated open on backing files. It introduces a new mode called FUSE_PASSTHROUGH_INO that requires one-to-one inode number mapping for attaching a backing file to a fuse inode. The patches also prepare the groundwork for future extensions, such as passthrough for additional operations like rename and unlink, and full subtree passthrough. The changes were tested using the passthrough_hp server and involve modifications to various FUSE-related files in the Linux kernel.
Joanne Koong (author) 2026-04-22 Neutral LLM
Author addressed concerns about the relationship between FOPEN_PASSTHROUGH and FUSE_PASSTHROUGH_RW_OPS, explaining that in FUSE_PASSTHROUGH_INO mode, FOPEN_PASSTHROUGH can be set with just one of read or write operations, but both are required for mmap. The author also acknowledged the need to enforce direct operations when a fuse inode has a backing inode and the server has opted out of passthrough for that file.
