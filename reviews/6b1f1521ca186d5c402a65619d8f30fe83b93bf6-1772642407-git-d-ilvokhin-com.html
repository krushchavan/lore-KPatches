[PATCH RFC 2/3] locking/percpu-rwsem: Extract __percpu_up_read_slowpath()

View on lore.kernel.org →
Submitted: 2026-03-04  ·  Last comment: 2026-03-05
Active on: 2026-03-10 • 2026-03-05 • 2026-03-04
Patch summary
The patch series aims to extract the __percpu_up_read_slowpath() function from the inline percpu_up_read() function in the locking/percpu-rwsem code, allowing for easier maintenance and modification of this critical path. This change is motivated by a potential binary size increase due to the addition of a tracepoint to an inlined function.
Peter Zijlstra 2026-03-04 Reviewed-by Neutral LLM
Recommended removing the 'extern' keyword from the function declaration for consistency with other declarations in the header file. Also suggested adding the '_slowpath' suffix to the function name, as it is used in the rtmutex code.
