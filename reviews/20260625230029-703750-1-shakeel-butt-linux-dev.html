[PATCH] mm/slub: serve slabobj_ext array from a strictly larger kmalloc cache

Patch summary
This patch fixes a kernel stack overflow issue caused by a recursive cycle between kmalloc-512 and kmalloc-1k slabs when freeing resources. The problem occurs because the obj_exts array, used for allocation profiling and memcg accounting, is allocated from the same cache as the slab it describes. To fix this, the patch changes the allocation to use a strictly larger cache than the one described by the slab, breaking the cycle and preventing the stack overflow.
