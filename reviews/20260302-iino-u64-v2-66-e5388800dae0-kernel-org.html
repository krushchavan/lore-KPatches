[PATCH v2 066/110] nfsd: replace PRIino with %llu/%llx format strings

View on lore.kernel.org →
Active on: 2026-03-02
Patch summary
This patch updates the NFS daemon's code to use explicit format strings instead of the deprecated PRIino macro, which is no longer needed since inode numbers are now u64. The changes affect several files in the nfsd directory and replace instances of PRIino with %llu or %llx format strings. This ensures that inode numbers are printed correctly as 64-bit values.
Jeff Layton (author) 2026-03-02 Neutral LLM
The author is re-posting the original patch, which suggests that they are not addressing any specific feedback or concerns from prior messages.

Now that i_ino is u64 and the PRIino format macro has been removed,
