Patch summary
This patch adds the SDEI (System Description Event Interface) as a last resort to stop CPUs in an arm64 system when normal methods fail. It signals event 0 at still-online CPUs, which runs crash_save_cpu() on the wedged context and parks them. This approach is chosen because an SDEI-stopped CPU never completes its event, making it less recoverable than a normal stop. The patch adds functionality to handle this scenario and includes changes to various kernel files.

Doug Anderson 2026-06-05 Inline Review Reviewed-by Needs Work LLM
Reviewer Doug Anderson noted that the SDEI cross-CPU NMI handler captures crash_save_cpu() state from the wedged context before parking the CPU, but this may not be sufficient to ensure recoverability since an SDEI-stopped CPU never completes its event. Reviewer Doug Anderson noted that the patch adds SDEI as a last resort for crash_smp_send_stop(), but not for regular smp_send_stop(), and suggested modifying smp_send_stop() to fall back to SDEI if sending the NMI failed, instead of adding this separate path. Reviewer Doug Anderson noted that passing a cpumask_t to the stack in crash_smp_send_stop() can lead to issues due to its potential size under certain CONFIG options, and suggested avoiding this by defining it as static.
