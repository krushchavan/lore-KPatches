[PATCH 2/3] iomap: decouple simple direct I/O reads from iomap_dio_rw

Patch summary
This patch decouples simple direct I/O reads from the iomap_dio_rw function, making it cleaner and more modular. It introduces a new function __iomap_dio_simple() to handle these reads, which properly accounts for synchronous cases by calling inode_dio_end() even on errors.
changfengnan 2026-07-23 Inline Review Reviewed-by Needs Work LLM
Reviewer changfengnan noted that the inode_dio_end() call in the synchronization scenario is missing a goto label to properly handle errors, and suggested adding a goto out_dio_end label before calling inode_dio_end().
