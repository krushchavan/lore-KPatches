Active on: 2026-04-09
Patch summary
This patch aims to eliminate busy-waiting in the shrink_dcache_parent function, which is responsible for shrinking dentry lists when processes exit. The current implementation can lead to stalls and crashes due to a use-after-free (UAF) bug. The patch likely addresses this issue by modifying the code path that handles submounts and namespace locking, ensuring that detach_mounts() and namespace_unlock() are not involved in the crash. The problem is related to a complex scenario involving multiple threads, list heads, and dentry traversal, making it difficult to reproduce and debug. The patch is part of an ongoing effort to improve kernel stability and performance.
