[PATCH v3 04/12] vfs: widen trace event i_ino fields to u64

View on lore.kernel.org →
Active on: 2026-03-04
Patch summary
This patch updates various VFS-layer trace event definitions to use u64 instead of ino_t/unsigned long for inode number fields, allowing for larger inode numbers and improving the accuracy of tracing events. The changes also update TP_printk format strings to match the widened field type and remove unnecessary casts. This patch is part of a larger series of updates to VFS-layer trace event definitions.
Jeff Layton (author) 2026-03-04 Neutral LLM
The author is addressing a concern about the use of u64 for inode numbers in trace events, specifically that it may not be necessary to change the type from unsigned long due to its size already being sufficient.
Show original comment 
Update VFS-layer trace event definitions to use u64 instead of
ino_t/unsigned long for inode number fields. Update TP_printk format
strings to use %llu/%llx to match the widened field type. Remove
now-unnecessary (unsigned long) casts since __entry->ino is already
u64.
