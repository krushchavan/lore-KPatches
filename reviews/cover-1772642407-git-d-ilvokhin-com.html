[PATCH RFC 0/3] locking: contended_release tracepoint instrumentation

View on lore.kernel.org →
Active on: 2026-03-05 • 2026-03-04
Patch summary
This patch series introduces a new tracepoint called contended_release that fires on the lock holder's side when a lock is released under contention. This allows for more accurate hold time estimation and provides the holder's stack at release time, which can differ from what is captured earlier in the locking process. The new tracepoint has minimal performance impact as it only affects slowpath unlock paths.
