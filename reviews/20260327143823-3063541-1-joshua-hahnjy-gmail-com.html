Patch summary
This patch optimizes memory hotplug by removing unnecessary checks and iterations, particularly in hot paths like shrink_memcg. It achieves this by directly checking for the first normal memory added to a node and setting a flag accordingly, eliminating the need for a notifier. Additionally, it simplifies the iteration process by accumulating node_normal_pages and clearing memory directly. The changes aim to improve efficiency without introducing bugs.
