[PATCH v3 09/12] ext4: widen trace event i_ino fields to u64

View on lore.kernel.org →
Submitted: 2026-03-04
Active on: 2026-03-04
Patch summary
This patch updates the ext4 trace events to use u64 instead of ino_t for inode numbers, allowing for wider range and more precise tracking of inode values. The changes affect several trace events, including ext4_other_inode_update_time, ext4_free_inode, and ext4_request_inode. The patch also updates the TP_printk format strings to match the new field type. This change is necessary due to the limitations of ino_t, which can lead to wraparound issues for large inode numbers.
Jeff Linux-f2fs-devel 2026-03-04 Neutral LLM
The reviewer noted that the patch does not update all instances of __field(ino_t, ...) to __field(u64, ...), specifically in the TP_printk format strings for ext4_evict_inode and ext4_nfs_commit_metadata.
