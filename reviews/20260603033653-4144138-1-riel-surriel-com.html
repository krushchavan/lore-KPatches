[PATCH v3 0/3] iova: use maple tree for O(log n) allocation

View on lore.kernel.org →

Active on: 2026-06-17
Patch summary
This patch series addresses performance issues in alloc_iova() caused by linear search, which can lead to soft lockups. It introduces a maple tree to index iova ranges, allowing alloc_iova() to have O(log n) complexity while maintaining similar memory usage. The patches also include self-tests for the iova code and were tested on various platforms.
