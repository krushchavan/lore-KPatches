Patch summary
This patch series improves large folio readahead and alignment for executable memory in the Linux kernel. The current implementation of exec_folio_order() requests readahead at an architecture-preferred folio order, but several issues prevent it from working optimally. These include the mmap_miss heuristic disabling readahead after 100 page faults, which is not triggered on arm64 with 64K pages. This patch series addresses these issues by aligning elf segments only when necessary, removing the preferred_exec_order() function, and changing the alignment order to HPAGE_PMD_ORDER if the number of pages in a vma is greater than or equal to HPAGE_PMD_NR. Additionally, it improves checking of p_vaddr and p_vaddr, removes unnecessary checks, and updates the patch series based on feedback from reviewers.

Jan Kara 2026-04-13 Reviewed-by Needs Work LLM
Reviewer Jan Kara expressed concerns that relying on PMD_SIZE for VMA alignment may lead to inefficient memory usage, particularly with small machines and large folios, suggesting capping the folio size at 2MB until a more sophisticated heuristic is developed.
Show original comment 
I'm not sure relying on PMD_SIZE will be too much for a VMA is a great
strategy. With 16k PAGE_SIZE the PMD would be 32MB large which would fit in
the .text size but already looks a bit too much? Mapping with PMD sized
folios brings some benefits but at the same time it costs because now parts
of VMA that would be never paged in are pulled into memory and also LRU
tracking now happens with this very large granularity making it fairly
inefficient (big folios have much higher chances of getting accessed
similarly often making LRU order mostly random). We are already getting
reports of people with small machines (phones etc.) where the memory
overhead of large folios (in the page cache) is simply too much. So I'd
have a bigger peace of mind if we capped folio size at 2MB for now until we
come with a more sophisticated heuristic of picking sensible folio order
given the machine size. Now I'm not really an MM person so my feeling here
may be just wrong but I wanted to voice this concern from what I can see...

								Honza
