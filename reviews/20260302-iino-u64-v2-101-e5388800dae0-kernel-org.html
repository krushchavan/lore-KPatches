[PATCH v2 101/110] fscrypt: replace PRIino with %llu/%llx format strings

View on lore.kernel.org →
Active on: 2026-03-02
Patch summary
This patch replaces the use of PRIino format macro in fscrypt code with explicit %llu and %llx format strings, as i_ino is now a u64 type. This change is necessary because PRIino has been removed from the kernel. The patch updates several files to reflect this change, ensuring that the code remains consistent and accurate.
Jeff Layton (author) 2026-03-02 Neutral LLM
Author acknowledged that the PRIino format macro has been removed and confirmed that all uses in fscrypt should be replaced with concrete format strings, specifically %llu/%llx for u64 values.
Show original comment 
Now that i_ino is u64 and the PRIino format macro has been removed,
replace all uses in fscrypt with the concrete format strings.
