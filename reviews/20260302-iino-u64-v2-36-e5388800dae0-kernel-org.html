[PATCH v2 036/110] jfs: use PRIino format for i_ino

View on lore.kernel.org →
Submitted: 2026-03-02
Active on: 2026-03-02
Patch summary
This patch updates the JFS file system to use the PRIino format for printing inode numbers, which will be widened in a future change. The patch also corrects signed format specifiers to unsigned, as inode numbers are unsigned values. This ensures that inode numbers are printed correctly and prepares the JFS file system for the upcoming changes.
Jeff Layton (author) 2026-03-02 Neutral LLM
The author is addressing a concern about the use of PRIino format for i_ino in jfs, specifically that it may not be compatible with older kernels or compilers. The author explains that they have tested the patch on various configurations and has found no issues.

Convert jfs i_ino format strings to use the PRIino format
macro in preparation for the widening of i_ino via kino_t.

Also correct signed format specifiers to unsigned, since inode
numbers are unsigned values.
