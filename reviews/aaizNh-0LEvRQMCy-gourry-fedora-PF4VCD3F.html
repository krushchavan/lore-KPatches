[PATCH] ACPI: NUMA: Only parse CFMWS at boot when CXL_ACPI is on

View on lore.kernel.org →
Submitted: 2026-03-05
Active on: 2026-03-05
Patch summary
This patch modifies the ACPI NUMA code to only parse CFMWS (Compact Firmware Memory Window Specification) at boot when CXL_ACPI is enabled. This change prevents the loss of NUMA structure information, which would otherwise be lost if CXL_ACPI is built into the kernel. The approach taken is to conditionally create these structures based on the presence of CXL_ACPI.
