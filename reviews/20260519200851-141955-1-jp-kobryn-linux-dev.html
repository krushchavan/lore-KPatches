Patch summary
This patch caps the compact_gap() function at COMPACT_CLUSTER_MAX to prevent kswapd from getting stuck in high-order reclaim mode when compaction is not suitable, leading to earlier fallback to order-0 balancing and waking up kcompactd for the original high-order request. This change addresses an issue where compaction_suitable() remains false due to the effective threshold being too low, causing kswapd to keep reclaiming in high-order mode. The patch aims to improve system performance by allowing compaction to become suitable sooner and reducing unnecessary high-order reclaim efforts.
