Patch summary
This patch updates the deferred split shrinker to use list_lru, which properly sets up per-node and per-cgroup lists. This change improves handling of cgroups on NUMA systems by avoiding splitting large pages across nodes. The patch also streamlines list operations and reclaim walks, and adds calls to create node heads for cgroups when splittable pages are created. Additionally, it introduces helpers to the list_lru API for caller-side locking. The deferred split queue lock is replaced with a more efficient approach using list_lru's locking mechanisms.
