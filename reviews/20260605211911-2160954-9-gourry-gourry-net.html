Patch summary
This patch adds a sysfs interface to control and query the state of an entire region of memory managed by the dax kmem driver. The new 'hotplug' attribute allows userspace to set the state of the memory region, which can be either online (in one of three possible zones) or unplugged. The patch also introduces a new 'unplugged' state that only applies to kmem/hotplug and provides atomic hot-unplug functionality across the entire device. This addresses a race condition in the previous implementation where external entities could operate on memory blocks while they were being removed, leading to failed hot-unplug operations.
