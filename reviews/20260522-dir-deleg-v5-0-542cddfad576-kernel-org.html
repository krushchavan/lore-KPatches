Active on: 2026-05-23
Patch summary
This patch series adds support for CB_NOTIFY callbacks in directory delegations, allowing clients to register interest in a directory and receive notifications about changes within it without losing their lease. The patches build on previous work in directory delegation and introduce new functionality to handle creates, unlinks, and renames, as well as updated directory attributes. The series includes various fixes and improvements, including proper error handling and tracepoint updates. The patchset has been tested with pynfs and is expected to be completed once a client-side implementation is available for testing.
Chuck Lever 2026-05-23 Reviewed-by Needs Work LLM
Reviewer noted that the patch does not properly handle the case where a client is making changes to a directory and receives a CB_NOTIFY callback for those same changes, which could lead to inconsistent state. reviewer noted that the patch does not properly handle the case where a dir delegation is released while still holding an inotify watch, which can lead to a use-after-free error
