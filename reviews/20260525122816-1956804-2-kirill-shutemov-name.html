Patch summary
This patch adds documentation to the Linux man-pages for the userfaultfd system call, specifically for the read-write protect mode (UFFDIO_REGISTER_MODE_RWP) introduced in Linux 7.2. The new feature allows user-space to receive page-fault notifications on any access (read or write) to a present page within a registered range, and provides options for asynchronous fault resolution. The patch updates the userfaultfd.2 man page to include descriptions of the new registration mode, pagefault flag, and VERSIONS line.
Alejandro Colomar 2026-05-25 Inline Review Reviewed-by Neutral LLM
Reviewer pointed out that the description of UFFDIO_REGISTER_MODE_RWP incorrectly states it traps every access to a present page, when in fact it only traps write accesses. The reviewer requested that the patch include more useful hunk contexts, citing documentation in the CONTRIBUTING file and CONTRIBUTING.d directory. The reviewer requested changes to the documentation formatting, specifically suggesting the use of \\[em] instead of \(em and semantic newlines to improve readability. The reviewer pointed out two minor formatting issues in the man-page documentation: 'by default' should be followed by a comma, and the word 'until' should be moved to the next line due to semantic newlines. The reviewer pointed out a minor grammar issue in the documentation, suggesting that 'user space' should be written as two words ('user-space') when used as a compound adjective. The reviewer suggested reformatting a paragraph in the man-page documentation for better readability, specifically recommending that each sentence or clause should be on its own line to avoid ambiguity and improve comprehension. Reviewer Alejandro Colomar suggested changing 'returns' to 'fails with/' in the documentation to accurately reflect that attempting to register both UFFDIO_REGISTER_MODE_RWP and UFFDIO_REGISTER_MODE_WP on the same range results in an error. The reviewer suggested renaming the 'read-write protect' mode to either 'read-write-protect' or 'read/write-protect' for consistency with the existing 'write-protect' mode.
