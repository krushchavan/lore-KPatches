[PATCH v2 00/21] fuse: extend passthrough to inode operations

Patch summary
This patch series extends the FUSE (Filesystem in Userspace) passthrough feature to support inode operations, including getattr, setattr, and readdir. The existing passthrough mode is modified to introduce a stricter mode called FUSE_PASSTHROUGH_INO, which requires one-to-one inode number mapping for attaching a backing file to a fuse inode. This allows for more efficient handling of inode operations and improves test results in certain scenarios.
Joanne Koong (author) 2026-05-18 Neutral LLM
Author acknowledged that server passing through GETATTR and opening files in caching mode is not currently supported, but proposed two options: either document the reason for requiring FOPEN_PASSTHROUGH or relax the limitation by allowing cached io mode.

> > > +Cached mode and passthrough mode are mutually exclusive. Attempting either

> > > +For directories, the same mechanism arbitrates between cached readdir
