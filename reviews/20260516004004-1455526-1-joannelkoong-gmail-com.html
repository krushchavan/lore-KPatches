Patch summary
This patch series extends the FUSE (Filesystem in Userspace) passthrough feature to support inode operations, including getattr, setattr, and readdir. The existing passthrough mode is modified to introduce a stricter mode called FUSE_PASSTHROUGH_INO, which requires one-to-one inode number mapping for attaching a backing file to a fuse inode. This allows for more efficient handling of inode operations and improves test results in certain scenarios.
Amir Goldstein 2026-05-16 Reviewed-by Needs Work LLM
Amir Goldstein noted that the patch introduces a new function fuse_kstat_to_attr() to convert kstat attributes to fuse_attr, but this function is not used in passthrough mode and instead calls vfs_getattr() on the backing inode. He expressed concern about the performance implications of always calling vfs_getattr(), potentially leading to unnecessary attribute updates.
