Patch summary
This patch introduces a new callback function called ->iomap_next() that replaces the existing ->iomap_begin() and ->iomap_end() functions. The ->iomap_next() function is responsible for finishing the previous mapping if needed and producing the next mapping, allowing performance-critical callers to inline the iteration with a fixed callback. A helper function called iomap_process() is also added to handle most of the logic required in an ->iomap_next() implementation. This change allows filesystems to be converted one at a time using the new ->iomap_next() function.
