[PATCH 2/3] mm/pagewalk: let folio_walk_start() run under the per-VMA lock

View on lore.kernel.org →
Submitted: 2026-06-16  ·  Last comment: 2026-06-19
Active on: 2026-06-19 • 2026-06-16
Patch summary
This patch introduces a new flag FW_VMA_LOCKED for the folio_walk_start() function, allowing it to run under the per-VMA lock instead of the mmap lock. This is done to reduce contention and improve performance for callers that only need to read a single page. The patch also adds documentation explaining the behavior and requirements of this new flag.
Lorenzo Stoakes 2026-06-19 Neutral LLM
Raised concerns about the patch, including unclear explanations of why RCU-freed page tables are required and how higher-level page tables would be freed. Also questioned the need for CONFIG_MMU_GATHER_RCU_TABLE_FREE.
