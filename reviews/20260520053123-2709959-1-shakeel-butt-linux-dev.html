Patch summary
This patch series addresses a performance regression caused by the introduction of per-node obj_cgroups, which led to thrashing in the per-CPU cache. The patches shrink the existing single-slot obj_stock_pcp fields and convert cached_objcg and nr_bytes into slot arrays to reduce cache footprint. A more general approach is taken to cache multiple objcgs per CPU using a multi-slot pattern, allowing sibling per-node objcgs to coexist without forcing a drain. This series aims to revert the temporary fix introduced earlier and improve performance on workloads with threads running on different NUMA nodes.
