[PATCH v2 042/110] overlayfs: use PRIino format for i_ino

View on lore.kernel.org →
Active on: 2026-03-02
Patch summary
This patch removes the kino_t typedef and PRIino format macro from the Linux kernel, replacing them with u64 type and format strings respectively. The changes are made throughout the tree, including in various file system-related files such as fs/dcache.c, fs/eventpoll.c, and include/linux/fs.h. The removal of these macros is necessary because i_ino has been widened to a 64-bit unsigned integer (u64), making the typedef and macro redundant.
Jeff Layton (author) 2026-03-02 Neutral LLM
The author is addressing a concern about the PRIino format macro being replaced with concrete format strings, specifically that the replacement for %PRIino in fs/inode.c was incorrect and should be %llx instead of %llu.
Show original comment 
Now that i_ino has been widened to u64, replace the kino_t typedef with
u64 and the PRIino format macro with the concrete format strings.

Replace the remaining PRIino uses throughout the tree, and remove the
typedef and #define from include/linux/fs.h. Change the i_ino field in
struct inode from kino_t to u64.
