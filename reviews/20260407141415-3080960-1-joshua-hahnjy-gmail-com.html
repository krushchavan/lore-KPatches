Patch summary
This patch fixes a naming issue in the weighted interleave auto sysfs attribute by directly passing a string 'auto' instead of the raw token, which was being expanded to __auto_type. This change prevents the sysfs entry from displaying as __auto_type and ensures it shows as 'auto'. The patch also bypasses an unnecessary permissions check for this specific file. The fix is based on a recent commit in the mm-new branch.
