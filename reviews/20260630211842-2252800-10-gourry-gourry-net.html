Patch summary
This patch adds a sysfs interface to enable atomic whole-device hotplug operations for DAX kmem devices. The new 'state' attribute allows devices to be transitioned between states such as unplugged, online, and offline_kernel or offline_movable in an atomic manner. This resolves a race condition issue where another entity could operate on memory blocks while the device is being removed, causing hot-unplug failures or deadlocks. The patch also updates the unbind process to prevent resource leaks and deadlock situations.

Cannot happen with a dev_group attribute. If probe succeeds then drvdata
is present. If probe fails, attribute never appears.

online_type with a different sentinel".

that only appears while the driver is attached would be something like:
