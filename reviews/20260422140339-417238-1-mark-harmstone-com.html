[PATCH] btrfs: don't force DIO writes to be serialized

Patch summary
This patch addresses an issue in the Btrfs file system where Direct I/O (DIO) writes are being serialized due to a missing flag that was set before the transition to the new mount API in 2023. The flag, SB_NOSEC, tells the VFS that the filesystem may have files without security xattrs, allowing optimizations. Without this flag, DIO writes to the same file will be locked exclusively, leading to reduced performance. This patch removes the serialization by not forcing DIO writes to be exclusive locks, resulting in a significant improvement in DIO throughput, as demonstrated by fio benchmark results showing a 59% increase.

It is. There's a big dropoff in DIO write performance in 6.8 that we 
never recovered from. I'm going to look into some sort of automated 
performance so this kind of thing can't happen casually.

This was on a VM with 8 cores and 8GB of RAM, with a real NVMe exposed 
through PCI passthrough. The figures for XFS and ext4 in comparison are 
both about ~3GB/s.

# cat go
#!/bin/bash
mkfs.btrfs -f /dev/nvme0n1
mount /dev/nvme0n1 /mnt/test
mkdir /mnt/test/nocow
chattr +C /mnt/test/nocow
fio /root/test.fio

# cat /root/test.fio
[global]
rw=randwrite
ioengine=io_uring
iodepth=64
size=1g
direct=1
startdelay=20
force_async=4
ramp_time=5
runtime=60
group_reporting=1
numjobs=32
time_based
disk_util=0
clat_percentiles=0
disable_lat=1
disable_clat=1
disable_slat=1
filename=/mnt/test/nocow/fiofile
[test]
name=test
bs=4k
stonewall
