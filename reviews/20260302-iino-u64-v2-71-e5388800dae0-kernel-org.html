[PATCH v2 071/110] afs: replace PRIino with %llu/%llx format strings

View on lore.kernel.org →
Submitted: 2026-03-02
Active on: 2026-03-02
Patch summary
This patch updates the AFS (Andrew File System) code to use the correct format strings for printing inode numbers, which are now u64 type. The PRIino macro has been removed and replaced with %llu/%llx format strings in all relevant places.
Jeff Layton (author) 2026-03-02 Neutral LLM
The author is addressing a concern about the format strings used in afs code, specifically that PRIino has been removed and replaced with %llu/%llx. The author's response is neutral as they are simply reiterating their original patch description without acknowledging any specific feedback or concerns.
