[PATCH v2 097/110] ubifs: replace PRIino with %llu/%llx format strings

View on lore.kernel.org →
Submitted: 2026-03-02
Active on: 2026-03-02
Patch summary
This patch updates the UBIFS (UBI File System) code to use the correct format strings for printing inode numbers, which are now u64 type. The PRIino macro has been removed and replaced with %llu/%llx format strings in all relevant files.
Jeff Linux-f2fs-devel 2026-03-02 Neutral LLM
The reviewer noted that the patch replaces PRIino format strings with %llu/%llx, but did not check if all affected code paths are lock-free and do not hold any locks when printing i_ino values.
Show original comment 
Now that i_ino is u64 and the PRIino format macro has been removed,
replace all uses in ubifs with the concrete format strings.

Signed-off-by: Jeff Layton <jlayton@kernel.org>
