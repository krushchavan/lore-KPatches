[PATCH 02/10] nfsd: drain callbacks and clear cl_cb_session

Patch summary
This patch fixes a bug in the NFS server where callbacks and session teardown could lead to a use-after-free situation. The issue arises when a DESTROY_SESSION is performed while an rpciod task still holds a reference to the callback session, causing it to be freed prematurely. To fix this, the patch introduces a two-stage drain mechanism in nfsd4_probe_callback_sync() that waits for any inflight callbacks to complete before releasing the final session reference. Additionally, clp->cl_cb_session is cleared unconditionally on error return from nfsd4_process_cb_update().

Chuck Lever 2026-05-29 Neutral LLM
Reviewer noted that the patch may introduce a denial-of-service (DoS) vulnerability due to NFS4ERR_DELAY responses from clients, which can cause NFSD callback done handlers to retry indefinitely and block foreground requests.
Show original comment 
Several NFSD callback done handlers retry indefinitely on
NFS4ERR_DELAY via rpc_delay(), so a client that keeps
replying DELAY leaves this per-client counter nonzero and
blocks the foreground CREATE/BIND/DESTROY_SESSION request
even though the callback no longer references the session
being torn down.

Although partly due to the way callbacks are structured
currently, this patch potentially introduces a client-
controlled DoS vector.
