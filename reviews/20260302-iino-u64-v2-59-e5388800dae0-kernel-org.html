[PATCH v2 059/110] vfs: use PRIino format for i_ino

View on lore.kernel.org →
Active on: 2026-03-02
Patch summary
This patch updates various parts of the Linux kernel to use the PRIino format macro for printing inode numbers (i_ino). The changes are made in four files: dcache.c, eventpoll.c, fserror.c, and pipe.c. The PRIino format is used instead of the old 0UL literal or direct formatting with %lx. This change aims to improve consistency and correctness in how inode numbers are printed throughout the kernel.
Jeff Layton (author) 2026-03-02 Neutral LLM
The author is addressing a concern about the use of PRIino format macro in various files, specifically in dcache.c where it was pointed out that the 0UL literal should be updated to (kino_t)0. The author's response is neutral as they are simply re-posting the original patch without any changes or explanations.
