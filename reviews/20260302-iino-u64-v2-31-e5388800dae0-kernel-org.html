[PATCH v2 031/110] hfs: use PRIino format for i_ino

View on lore.kernel.org →
Submitted: 2026-03-02
Active on: 2026-03-02
Patch summary
This patch updates the HFS file system to use the PRIino format for printing inode numbers, in preparation for the future widening of inode values from 32-bit to 64-bit via the kino_t type.
Jeff Layton (author) 2026-03-02 Neutral LLM
Author acknowledged that the PRIino format is used for i_ino in the patch, and it's a preparation for widening of i_ino via kino_t.
