[PATCH v2 063/110] f2fs: replace PRIino with %llu/%llx format strings

View on lore.kernel.org →
Submitted: 2026-03-02
Active on: 2026-03-02
Patch summary
This patch replaces the PRIino format macro with concrete format strings (%llu/%llx) in the f2fs file system, as i_ino is now a u64 type and PRIino has been removed. The changes are made across various files in the f2fs codebase to ensure consistency and correctness in formatting inode numbers. This patch addresses a technical debt issue that arose from the change in inode number type and format macro removal.
Jeff Layton (author) 2026-03-02 Neutral LLM
Author acknowledged that the patch needs to replace all uses of PRIino format macro with concrete %llu/%llx format strings, and confirmed that this has been done in the updated patch.
