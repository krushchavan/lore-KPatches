Patch summary
This patch replaces the exec_folio_order() function with a generic preferred_exec_order() function, which sets the readahead order to a minimum value based on the available memory. The existing code already naturally steps down allocation orders under memory pressure, making the explicit zone_watermark_ok() loop redundant. The new approach simplifies the code and achieves the same result by setting the readahead order to a fixed value for VM_EXEC cases.
Usama Arif (author) 2026-03-26 Neutral LLM
Author acknowledged that the explicit zone_watermark_ok() loop might be redundant, considering readahead_gfp_mask() already sets __GFP_NORETRY and page_cache_ra_order() falls back to order 0 allocation when not satisfied, and proposed a simpler approach by setting ra->order directly. Author acknowledges that the explicit zone_watermark_ok() loop might be redundant, proposes a simpler approach by setting ra->order to min(HPAGE_PMD_ORDER, ilog2(SZ_2M >> PAGE_SHIFT)) and suggests using gfp &= ~__GFP_RECLAIM for the VM_EXEC case.
