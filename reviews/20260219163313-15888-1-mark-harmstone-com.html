[PATCH] btrfs: add check in remove_range_from_remap_tree() for a NULL block group

View on lore.kernel.org →
Active on: 2026-02-19
Patch summary
The patch adds a check in the remove_range_from_remap_tree() function to handle a NULL block group, preventing potential segfaults and returning an error instead.
Filipe Manana 2026-02-19 Reviewed-by Positive LLM
Suggested using unlikely() to handle the unexpected NULL block group, and added a Fixes tag referencing the related patch in Linus' tree.
