[PATCH v1] iomap: fix invalid folio access when i_blkbits differs from I/O granularity

Patch summary
This patch addresses an issue where the iomap_read_folio_iter function fails to invalidate ctx->cur_folio when the IO granularity differs from the block size of the underlying filesystem or device, leading to potential double-put bugs and invalid folio access. The fix involves unconditionally invalidating ctx->cur_folio in this case. This patch also includes a test report and link to the original issue.
