[PATCH v3 0/1] writeback: don't block sync for filesystems with no data integrity guarantees

View on lore.kernel.org →
Submitted: 2026-03-20
Active on: 2026-03-20
Patch summary
This patch prevents the sync operation from blocking for file systems that do not guarantee data integrity. It achieves this by skipping the check in sync_inodes_one_sb() and instead checking in sync_inodes_sb(). This ensures that even if a file system does not ensure data integrity, it will still allow the flusher threads to run during sync operations.
