[PATCH v4 3/4] cleanup: Annotate guard constructors with __nonnull()

View on lore.kernel.org →
Submitted: 2026-05-23
Active on: 2026-05-23
Patch summary
This patch series aims to annotate guard constructors in the Linux kernel with __nonnull() to ensure that the compiler warns when NULL is statically known to be passed. The patch adds a __nonnull() macro in compiler_attributes.h and updates several files, including cleanup.h, to use this macro.
