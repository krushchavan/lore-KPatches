Patch summary
This patch fixes stale batch updates in the mm/mglru code after a memory cgroup (memcg) is reparented. The issue arises when a memcg is being destroyed and its children are being moved to another parent, causing stale LRU updates. The fix involves checking if the current memcg is being destroyed before updating its LRU batch.
