[PATCH v2 034/110] isofs: use PRIino format for i_ino

View on lore.kernel.org →
Submitted: 2026-03-02  ·  Last comment: 2026-03-03
Active on: 2026-03-02
Patch summary
This patch updates the isofs (ISO9660) filesystem code to use the PRIino format macro for printing inode numbers, which are unsigned values. This change prepares the code for the future widening of inode numbers via the kino_t data type. The patch also corrects signed format specifiers in several print statements to match the unsigned nature of inode numbers.
★ Jeff Layton (author) 2026-03-02 Neutral LLM
The author is addressing a concern about the use of PRIino format for i_ino in the patch, specifically that it may not be compatible with older kernels or systems without support for the kino_t type. The author acknowledges this issue and notes that they will need to revisit the patch to ensure compatibility.
