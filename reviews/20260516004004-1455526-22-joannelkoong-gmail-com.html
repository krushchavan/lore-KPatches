Patch summary
This patch adds documentation to the Linux kernel's FUSE (Filesystem in Userspace) module, specifically about extended passthrough mode (FUSE_PASSTHROUGH_INO). This new feature allows for inode operations and directory operations to be passed through from the user space to the kernel, with a one-to-one mapping between fuse inode numbers and backing file inode numbers. The documentation explains how to enable this feature, including steps for negotiating the capability during FUSE_INIT, registering backing files, and handling requests in the daemon.
Amir Goldstein 2026-05-16 Inline Review Reviewed-by Neutral LLM
The reviewer pointed out that the documentation incorrectly states that files opened with FOPEN_DIRECT_IO deny entry into both caching and passthrough IOMODE, when in fact they only deny entry into caching. Reviewer noted that the requirement for FOPEN_PASSTHROUGH when opening files in caching mode without doing IO passthrough could be perplexing to users, and suggested either documenting this limitation or relaxing the restriction.
