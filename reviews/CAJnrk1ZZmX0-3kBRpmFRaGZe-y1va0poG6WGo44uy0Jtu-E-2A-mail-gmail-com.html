[PATCH] fuse: fix inode initialization race

View on lore.kernel.org →
Submitted: 2026-03-26
Active on: 2026-03-26
Patch summary
This patch fixes a race condition in the FUSE (Filesystem in Userspace) kernel module related to inode initialization. The issue arises when a new inode is created and its attributes are changed immediately after, causing the truncate_pagecache() and invalidate_inode_pages2() functions to be bypassed due to incorrect conditions. To fix this, the patch ensures that these functions are called even for new inodes by modifying the fuse_change_attributes_i() function.

Joanne Koong (author) 2026-03-26 Neutral LLM
Author addressed a concern that fuse_change_attributes_i() takes the fi lock instead of the inode lock, and explained why it's safe not to take the inode lock in this function because truncate_pagecache() and invalidate_inode_pages2() are gated by conditions that will always be false for I_NEW inodes. Author addressed a concern that the condition in fuse_change_attributes_i() could trigger an inode initialization race, explaining that fuse_iget() sets inode->i_size to attr->size before calling fuse_change_attributes_i(), making the condition impossible to trigger.

We never call into truncate_pagecache() or invalidate_inode_pages2()
from fuse_change_attributes_i():
 truncate_pagecache() is gated by oldsize != attr->size:
    for I_NEW inodes, fuse_init_inode() sets inode->i_size to
attr->size, which means "oldsize != attr->size" is alwyas going to be
false

  invalidate_inode_pages2() is gated by oldsize != attr->size and
"!timespec64_equal(&old_mtime, &new_mtime)" (where new_mtime  ses the
mtime values from attr):
    for I_NEW inodes, fuse_init_inode() calls inode_set_mtime(inode,
attr->mtime, attr->mtimensec);, which means
"!timespec64_equal(&old_mtime, &new_mtime)" is always going to be
false

For a new inode, oldsize is attr->size, not 0. The condition never triggers.

fuse_iget() calls fuse_init_inode() (which sets inode->i_size to
attr->size) before it calls fuse_change_attributes_i().
