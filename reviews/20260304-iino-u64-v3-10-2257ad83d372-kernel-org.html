[PATCH v3 10/12] f2fs: widen trace event i_ino fields to u64

View on lore.kernel.org →
Submitted: 2026-03-04
Active on: 2026-03-04
Patch summary
This patch updates the f2fs trace events to use u64 instead of ino_t for the inode number field, allowing it to handle larger inode numbers. The TP_printk format strings are also updated to match the new field type. This change is necessary because the current implementation may overflow or lose precision when dealing with large inode numbers.
Jeff Layton (author) 2026-03-04 Neutral LLM
The author is addressing a concern about the change from ino_t to u64 in trace events, specifically that it may cause issues due to the use of %lu format specifier in TP_printk strings. The author has updated the patch to use %llu instead, which should fix the issue.
Show original comment 
In trace events, change __field(ino_t, ...) to __field(u64, ...)
and update TP_printk format strings to %llu/%llx to match the
widened field type.

Signed-off-by: Jeff Layton <jlayton@kernel.org>
