[PATCH v4 3/5] mm: convert compaction to zone lock wrappers

Patch summary
This patch series converts the compaction code to use zone lock wrappers, replacing direct access to spinlocks. This change is necessary because the zone lock operations are now wrapped and cannot be accessed directly by compact_lock_irqsave(). The patch splits the helper into two functions: compact_zone_lock_irqsave() and compact_lruvec_lock_irqsave(), duplicating shared logic for the two call sites.

> shared logic. As there are only two call sites and both statically
> know the lock type, this avoids introducing additional abstraction
> or runtime dispatch in the compaction path.
>
