Patch summary
This patch modifies the Btrfs file system to handle ENOMEM errors more gracefully when inserting directory items. Instead of aborting transactions under memory pressure, it converts newly-created inodes into orphans and returns -ENOMEM to userspace. This change affects create(), mkdir(), mknod(), symlink(), and link() operations. The approach taken involves modifying the btrfs_add_link() function to handle ENOMEM errors and adding orphan handling in btrfs_create_new_inode().

For btrfs_alloc_path(), you can just use on-stack memory for that.

That's the practice we utilize for btrfs-progs, and it's still 
acceptable for kernel, since that structure is only 112 bytes.

We should not switch all btrfs_path to on-stack ones, but for critical 
ones that memory allocation can lead to trans abort, I'd say it's 
definitely worth the extra on-stack memory usage.
