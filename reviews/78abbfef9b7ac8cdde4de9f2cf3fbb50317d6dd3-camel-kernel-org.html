[PATCH v2 001/110] vfs: introduce kino_t typedef and PRIino format macro

View on lore.kernel.org →
Active on: 2026-03-03
Jeff Linux-f2fs-devel 2026-03-03 Inline Review Neutral Heuristic
Mostly it's to allow 'PRIino "x"'. There are a number of places that (for whatever reason) print the inode number in hex. I don't want to change those. There are also some places that print it as a signed value (PRIino "d"). I suspect most of those are bugs, or just holdovers from a simpler time when we didn't worry so much about the signedness of inode numbers. I fixed a few of those in the context of this series, fwiw.
