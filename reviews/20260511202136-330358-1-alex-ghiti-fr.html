Patch summary
This patch series introduces per-memcg-per-node kernel memory accounting to associate memcg and physical memory usage. It builds on recent changes to obj_cgroup, which ties object groups to their NUMA node. The patches split kmem charging and accounting for percpu allocations, and convert other kmem users such as slab and zswap to use the new per-node accounting. This allows for more accurate tracking of kernel memory usage at a per-node level.
