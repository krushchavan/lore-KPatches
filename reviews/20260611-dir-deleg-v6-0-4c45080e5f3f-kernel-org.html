Patch summary
This patch series adds support for CB_NOTIFY callbacks in directory delegations, allowing clients to register interest in a directory and receive notifications about changes within it without losing its lease. The patches include fixes for various issues, such as RCU handling and encoding failures, and have been tested with pynfs but require a complete client-side implementation for final testing. The series builds on previous work on directory delegation and adds support for creates, unlinks, and renames, as well as sending updated directory attributes in notifications.
Jeff Layton (author) 2026-06-11 Positive LLM
The author addressed a concern about the lease breaker ownership check in nfsd_breaker_owns_lease() by adding a quick check for nfsd leases, which will have their fl_lmops set to nfsd_lease_mng_ops. The patch adds a new constant struct lease_manager_operations and updates the function to return false if the lease manager operations are not equal to nfsd_lease_mng_ops.
