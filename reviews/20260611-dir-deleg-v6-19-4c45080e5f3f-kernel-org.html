[PATCH v6 19/20] nfsd: track requested dir attributes

View on lore.kernel.org →
Submitted: 2026-06-12
Active on: 2026-06-12
Patch summary
This patch introduces a new feature to track the union of requested and supported directory attributes in the delegation, which will be used later to ensure that only relevant attributes are encoded when sending updates.
Chuck Lever 2026-06-12 Neutral LLM
Identified a potential issue with the patch's impact on bisectability, specifically that it allows clients to request NOTIFY4_CHANGE_DIR_ATTRS without actually being able to send dir-attribute notifications.

Bisectability: After this patch is applied, a client that requests
NOTIFY4_CHANGE_DIR_ATTRS now gets that bit echoed in gddr_notification,
but the callback path still only maps/encodes add, remove, and rename
notifications (nfsd_notify_to_ignore(), nfsd_fsnotify_recalc_mask(),
and nfsd4_encode_notify_event() have no dir-attr case). That lets the
server grant a directory delegation while promising dir-attribute
CB_NOTIFYs it cannot send until the follow-up support lands, so this
bit should not be advertised in this patch.
