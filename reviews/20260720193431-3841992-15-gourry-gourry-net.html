[PATCH v5 14/36] mm/damon: skip private node memory in DAMON migration and pageout

Active on: 2026-07-21 • 2026-07-20
Patch summary
This patch series modifies the DAMON (Data Monitoring) kernel module to skip private node memory during migration and pageout operations. The changes aim to prevent potential issues that may arise from reclaiming or migrating private node memory, which is not intended for such operations.
SJ Park 2026-07-21 Neutral LLM
Raised concerns about the language used in the patch, specifically the term 'by default', which may imply that private node memory can be reclaimable in certain situations. Asked for clarification on what happens if private node memory is paged out.
