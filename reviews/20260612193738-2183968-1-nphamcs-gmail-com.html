Patch summary
This patch series introduces Virtual Swap Space (Swap Table Edition), which aims to address the limitations of current swap design by decoupling swap entries from physical backing slots. This allows for more efficient use of disk space, especially in scenarios where zswap is used. The changes include rebasing to a newer mm-unstable tip, fixing assorted issues, and simplifying code. Additionally, the patch series introduces new features such as deferring memcg_table allocation on physical clusters until the first vswap-backing slot installs and widening swap_info_struct->max and ->pages to support up to 8 PB of swap space.
