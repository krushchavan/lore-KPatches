Patch summary
This patch updates the deferred split shrinker to use list_lru, which properly sets up per-node and per-cgroup lists. This change addresses a suboptimal handling of cgroups in the deferred split queue, where large pages on other nodes were being split unnecessarily. The new approach streamlines list operations and reclaim walks, and is used by other major shrinkers already. The patch also adds calls to create node heads for the cgroup at once when splittable pages are created, eliminating the need for special care in migration code between nodes.
