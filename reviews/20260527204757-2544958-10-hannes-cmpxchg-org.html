Patch summary
This patch updates the deferred split shrinker to use list_lru, which properly sets up per-node and per-cgroup lists. This change addresses a suboptimal handling of cgroups in the deferred split queue, where large pages on other nodes were being split unnecessarily. The new approach streamlines list operations and reclaim walks, and is used by other major shrinkers already. The patch also adds calls to create node heads for the cgroup at once when splittable pages are created, eliminating the need for special care in migration code between nodes.

From: Lance Yang <lance.yang@linux.dev>
Date: Tue, 9 Jun 2026 10:56:45 +0800
Subject: [PATCH] mm: prepare deferred split metadata only for anon swapin

__swap_cache_alloc() prepares deferred split metadata for large swapcache
folios.

That also covers shmem swapin, because shmem_swap_alloc_folio() can call
swapin_sync() with a large order[1]. But shmem folios are not queued on
the deferred split queue, so preparing the metadata doesn't buy us
anything there.

So let's limit it to anon swapin.

[1] https://lore.kernel.org/all/20260601132135.14272-1-lance.yang@linux.dev/
