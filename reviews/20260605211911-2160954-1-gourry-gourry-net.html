Patch summary
This patch series adds atomic whole-device hotplug control via sysfs, allowing for transitions to be atomic across entire device ranges. The patch addresses a race condition between offlining and removing memory blocks, which was discussed in the LPC2025 device memory sessions. It introduces a 'hotplug' attribute and updates mm and dax plumbing to support it. The state names mirror the per-block memoryX/state ABI with modifications for atomic whole-device control. The patch also includes mm preparation changes, such as adding helpers for online type and offline+remove of multiple ranges under a single lock.
