Patch summary
This patch adds a sysfs interface to enable atomic whole-device hotplug operations for DAX kmem devices. It introduces a new 'state' attribute that allows the device to be transitioned between different states, including unplugged, online, and offline. The patch also includes changes to handle legacy auto_online_blocks=offline policy and prevents deadlock during unbind operation. The atomic whole-device hotplug operation is achieved through a single call to dax_kmem_do_hotremove(), which collects all added ranges and offlines/removes them in one operation.
Gregory Price (author) 2026-06-25 Neutral LLM
The author acknowledges that unbind is a violent operation and cannot fail, but still wants to prevent deadlocks by leaking resources if the device is online at unbind time. The author plans to reinstate legacy behavior when the state is not UNPLUGGED.
