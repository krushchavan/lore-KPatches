Patch summary
This patch adds an interface for swap tier selection in the Linux kernel's memory control system, addressing concerns about accidental overcommitting of resources when using multiple workloads and tiers. The approach taken is to provide a default 'reasonable' scaling that users can override if needed, rather than allowing opt-in/opt-out for individual tiers. This change aims to simplify the configuration process while preventing potential resource overcommit issues.
Joshua Hahn (author) 2026-06-23 Needs Work LLM
Author acknowledges a concern about system-critical processes being demoted in tiered memcg limit model and agrees that they should be prioritized. Author addressed Yosry Ahmed's concern about accidental overcommitting in swap tier selection by proposing to make swap tiers a no-op unless users opt-in to setting limits themselves, and suggested changing the memory tiering series to not present default settings for tiered limits.

Reviewer Yosry Ahmed noted that there may be use cases where some memcgs are excluded from certain memory tiers, such as limiting sensitive workloads to the top tier or vice versa. Reviewer Yosry Ahmed noted that both presenting a default value for swap tier selection and allowing users to manually set limits can lead to overcommitting issues, and suggested removing the default value so that 'max' is the default for all tiers, allowing userspace to set custom limits if needed.
