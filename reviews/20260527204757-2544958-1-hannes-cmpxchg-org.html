Patch summary
This patch series, version 5, aims to switch the THP (Transparent Huge Pages) shrinker to use list_lru instead of the current implementation. The changes include fixing a long-standing issue in set_shrinker_bit() where the bit can be set on a dying child memcg instead of the ancestor that received the item. Other changes include adapting to the new lock_list_lru_of_memcg() signature, propagating folio_memcg_alloc_deferred() failure, and fixing a use-after-free bug in deferred_split_isolate(). The patches have been rebased onto the latest mm-unstable tree.

Lorenzo Stoakes 2026-06-09 Inline Review Reviewed-by Neutral LLM
The reviewer pointed out that the current implementation of folio_memcg_alloc_deferred() does not filter shmem swapin, which is not a problem because shmem folios are not queued on the deferred split queue. However, they noted that a folio can be anon for a non-shmem file-backed VMA, and suggested adding vma_is_anonymous(vmf->vma) to the condition.
