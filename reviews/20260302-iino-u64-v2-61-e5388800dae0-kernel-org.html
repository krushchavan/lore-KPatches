[PATCH v2 061/110] ext4: replace PRIino with %llu/%llx format strings

View on lore.kernel.org →
Submitted: 2026-03-02  ·  Last comment: 2026-03-03
Active on: 2026-03-02
Patch summary
This patch replaces the use of PRIino format macro in ext4 code with concrete format strings %llu and %llx, which are suitable for u64 type inode numbers. This change is necessary because PRIino has been removed from the kernel. The patch updates various files in the ext4 directory to reflect this change.
★ Jeff Layton (author) 2026-03-02 Neutral LLM
The author is addressing a concern about the use of %llu/%llx format strings in ext4 code, specifically that they are not used consistently throughout the codebase. The author agrees to restructure the code in v2 to ensure consistency and fix any issues.
Show original comment 
Now that i_ino is u64 and the PRIino format macro has been removed,
replace all uses in ext4 with the concrete format strings.

Signed-off-by: Jeff Layton <jlayton@kernel.org>
