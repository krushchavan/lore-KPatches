[PATCH 0/4] arm64: cross-CPU NMI via SDEI

View on lore.kernel.org →

Active on: 2026-06-05
Patch summary
This patch series adds a new mechanism for delivering NMI (Non-Maskable Interrupts) across CPUs on arm64 architecture, using the SDEI (System Description Event Infrastructure) protocol. The goal is to enable features like hard-lockup detection and crash reporting without incurring significant performance overhead. Currently, two existing mechanisms are available: pseudo-NMI, which has a hot-path cost, and FEAT_NMI, which is not present in deployed silicon. This new approach uses SDEI events to deliver interrupts, allowing for efficient interrupt masking and reducing the impact on system performance.
