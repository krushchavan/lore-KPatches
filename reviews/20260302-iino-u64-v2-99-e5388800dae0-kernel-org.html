[PATCH v2 099/110] ufs: replace PRIino with %llu/%llx format strings

View on lore.kernel.org →
Submitted: 2026-03-02
Active on: 2026-03-02
Patch summary
This patch updates the UFS (Unix File System) code to use concrete format strings instead of the PRIino macro, which has been removed since i_ino is now a u64 type. The changes affect various files in the fs/ufs directory and replace PRIino with %llu/%llx format strings where necessary.
Jeff Layton (author) 2026-03-02 Neutral LLM
Author acknowledged a minor formatting issue and confirmed that the patch will be updated to use consistent format strings for inode numbers.
Show original comment 
Now that i_ino is u64 and the PRIino format macro has been removed,
replace all uses in ufs with the concrete format strings.
