Patch summary
The patch addresses an issue where the kernel fails to recognize a client-side upcall (cld) and instead falls back to running a usermodehelper upcall, which is unexpected behavior. The problem is likely due to warnings about creating pipes in rpc_pipefs, as indicated by dmesg logs. This patch aims to resolve this issue by fixing the underlying cause of the kernel's failure to recognize the cld upcall.
