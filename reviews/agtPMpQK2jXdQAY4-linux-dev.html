Patch summary
This patch addresses a regression in the memcg (memory cgroup) code, specifically related to the obj_stock_pcp structure and its handling of object caching. The patch modifies the obj_stock_pcp structure to cache multiple obj_cgroups instead of just one, which improves performance by reducing cache misses. It also introduces a new NR_OBJ_STOCK define that determines the number of cached obj_cgroups. Additionally, the patch includes changes to the obj_stock_pcp initialization and locking mechanisms to support the new caching behavior.
Oliver Sang 2026-05-19 Inline Review Reviewed-by Needs Work LLM
Reviewer noted that the obj_stock_pcp structure has been modified to fit within a single 64-byte cache line on non-debug 64-bit builds, but this change may not be sufficient as it only accounts for one CPU consuming or refilling against five different objcgs, and requested further investigation into the impact of multiple CPUs accessing the same slot. Reviewer noted that the patch v2 (8da1b1ea43) can recover performance when applied on top of 01b9da291c, but since it's out-of-date, they won't provide a full comparison. Reviewer Oliver Sang noted that the patch is significantly different from v2 or v3 and requested confirmation on whether to test v3 instead.
