Patch summary
This patch introduces a new feature to the Linux kernel's NUMA (Non-Uniform Memory Access) subsystem, allowing for the creation of 'standby' nodes at boot time. These standby nodes can be claimed by drivers at runtime, enabling them to place hotplugged memory on distinct NUMA nodes without requiring BIOS-assigned proximity domains. The patch includes a new exclusive node pool with claim and release interfaces, as well as modifications to the numa_init_standby_nodes() function to rebuild the NUMA distance table for standby nodes. This feature is made possible by a new boot parameter 'numa=standby=<N>', which specifies the number of standby nodes to create. The patch also addresses potential issues with dynamic node addition and phys_to_target_node() ambiguity.
