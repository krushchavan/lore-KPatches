Patch summary
This patch addresses a potential race condition in the zswap system, which can occur when zswap writeback is enabled and a PMD swap entry is being loaded. The issue arises when one CPU thread is loading a large-folio swap-in while another thread is splitting a PTE (page table entry) and adding it to the zswap cache. To resolve this, the patch adds range lookup for large-folio swapin, ensuring that if the PMD swap entry is partially loaded, the correct action is taken by rejecting the PMD order read and falling back to per-page loading.
