[PATCH v2 068/110] nilfs2: replace PRIino with %llu/%llx format strings

View on lore.kernel.org →
Submitted: 2026-03-02
Active on: 2026-03-02
Patch summary
This patch updates the nilfs2 file system to use explicit format strings instead of the PRIino macro, which has been removed since inode numbers are now u64 type. The changes affect various files in the nilfs2 codebase and replace instances of PRIino with %llu or %llx format strings as appropriate. This is a part of a larger series of patches that aim to modernize the nilfs2 file system.
Jeff Layton (author) 2026-03-02 Neutral LLM
The author is addressing a concern about inconsistent format strings for inode numbers in error messages, which were previously using the PRIino macro and are being replaced with %llu/%llx format strings. The author agrees that the changes look good and confirms that all uses of PRIino have been removed.
Show original comment 
Now that i_ino is u64 and the PRIino format macro has been removed,
replace all uses in nilfs2 with the concrete format strings.
