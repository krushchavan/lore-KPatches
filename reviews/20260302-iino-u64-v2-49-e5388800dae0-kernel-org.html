[PATCH v2 049/110] security: use PRIino format for i_ino

View on lore.kernel.org →
Submitted: 2026-03-02
Active on: 2026-03-02
Patch summary
This patch converts security-related code to use the PRIino format macro for printing inode numbers, which are unsigned values. This is a preparatory step for widening the inode number type via kino_t. The changes affect six files in the kernel's security subsystem and correct signed format specifiers to match the unsigned nature of inode numbers.
Jeff Layton (author) 2026-03-02 Neutral LLM
The author is addressing a concern about the use of PRIino format for i_ino, specifically in audit log messages. The author explains that they have already converted all relevant code to use PRIino and provides patch diffs as evidence.
