[PATCH 2/2] x86/tdx: Fix zero-extension for 32-bit port I/O

View on lore.kernel.org →
Submitted: 2026-03-31
Active on: 2026-03-31
Patch summary
This patch series addresses a bug in the TDX (Trusted Domain Extensions) port I/O handling on x86 architecture. The current implementation of handle_in() only masks the lower 32 bits, preserving the upper 32 bits of RAX when a 32-bit port IN instruction is emulated. This patch updates handle_in() to zero out the entire RAX register when the I/O size is 4 bytes, ensuring correct zero-extension for 32-bit operations.
