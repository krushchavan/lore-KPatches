Patch summary
This patch adds support for zero-copy operations over io-uring in the fuse file system. It eliminates memory copies between userspace, the kernel, and the fuse server by allowing the server to directly access client pages or page cache folios without copying data through an intermediary buffer. The patch introduces a new feature that requires CAP_SYS_ADMIN and FUSE_URING_REGISTERED_BUFFERS capabilities, which enables the server to register relevant pages into its io-uring registered buffer table. This allows for more efficient read/write operations by reducing memory copies. The patch also includes documentation of the request flow for zero-copy write path.

The userspace FIXED_READ call (eg set up usually through the liburing
api io_uring_prep_read_fixed() [1]) takes a buf_index, not an address.
Userspace would pass the ent->zero_copy_index as the buf index.
ent->zero_copy_index corresponds to the fixed buffer where userspace
told the kernel (at sqe registration time through the "uint16_t
zero_copy_index" init param) to place the zero-copied pages for that
sqe.

Joanne Koong (author) 2026-06-17 Neutral LLM
The author addressed a concern about the calculation of the offset into the fixed buffer for zero-copied registered buffers, explaining that io-uring internally sets the buffer's uaddr as 0 and the liburing API computes the offset by subtracting imu->ubuf from the provided address. The author did not indicate any plans to change their code.
Show original comment 
For zero-copied registered buffers, the "address" you pass in is the
raw offset into the buffer (eg if you want to read into the
zero-copied buffer starting at offset 4k in the buffer, you would call
io_uring_prep_read_fixed(sqe, fd, (void *)4096, len, file_off,
buf_index);).

On the kernel side, fuse uring calls io_buffer_register_bvec() to
place the application pages or page cache pages into the slot for the
registered buffer [1]. io-uring internally sets that buffer's uaddr as
0 (imu->ubuf = 0 in io_kernel_buffer_init() [2]) since the pages are
supplied kernel-side. The liburing io_uring_prep_read_fixed() api will
call into the kernel io_read_fixed() -> io_init_rw_fixed() ->
io_import_reg_buf() -> io_import_fixed(). In io_import_fixed(), the
offset into the fixed buffer is computed with "offset = buf_addr -
imu->ubuf" [3] where buf_addr is the u64 representation of the "void
*addr" that was passed into io_uring_prep_read_fixed().

Joanne
