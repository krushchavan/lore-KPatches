Patch summary
This patchset adds support for netlink upcalls to the mountd/exportd services, replacing the outdated /proc-based interfaces used by these services. The new netlink-based interfaces provide a more modern and flexible way to handle cache notifications between the kernel and userland. This change is necessary due to limitations in the current implementation, including difficulty extending it to support new export options and reliance on /proc which may not be desirable in container environments. The patchset includes 14 patches that modify various kernel components, including nfsd, sunrpc, and netlink, to implement the new interfaces.
Jeff Layton (author) 2026-03-16 Neutral LLM
The author addressed a concern about the naming consistency of the upcall functions by renaming sunrpc_cache_pipe_upcall() to sunrpc_cache_upcall(). The change is reflected in multiple files, including fs/nfsd/export.c and net/sunrpc/cache.c. The author confirmed that this change will be part of the upcoming patch version.

to allow callers to count and snapshot the pending upcall request list

Show original comment 
The auth.unix.ip and auth.unix.gid caches live in the sunrpc module,
so they cannot use the nfsd generic netlink family. Create a new
"sunrpc" generic netlink family with its own "exportd" multicast
group to support cache upcall notifications for sunrpc-resident
caches.

Define a YAML spec (sunrpc_cache.yaml) with a cache-type enum
(ip_map, unix_gid), a cache-notify multicast event, and the
corresponding uapi header.

Implement sunrpc_cache_notify() which mirrors the nfsd_cache_notify()
pattern: check for listeners on the exportd multicast group, build
and send SUNRPC_CMD_CACHE_NOTIFY with the cache-type attribute.

Register/unregister the sunrpc_nl_family in init_sunrpc() and
cleanup_sunrpc().
