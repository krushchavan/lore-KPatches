Patch summary
This patch addresses performance issues caused by the IOCB_DONTCACHE flag, which triggers aggressive flushing of dirty pages on every write operation. The problem arises when multiple writers contend for writeback submission, leading to severe serialization and reduced throughput. To fix this, a new function filemap_dontcache_writeback_range() is introduced, which uses two rate-limiting mechanisms: skip-if-busy (checking if writeback is already in progress) and proportional cap (limiting the number of pages flushed). These mechanisms prevent excessive flushing and contention between concurrent writers. The patch replaces the existing filemap_flush_range() call with the new function in generic_write_sync().
