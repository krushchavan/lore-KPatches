Ongoing Submitted 2026-04-23 🔥 TODAY [PATCH RFC] sunrpc: hardcode pool_mode to pernode, remove other modes Needs Work LLM (per-reviewer) Under Review

This patch removes the SVC_POOL_* enum and mode selection heuristic, simplifying the pool map functions to always use the pernode path. The percpu mode is removed as it created excessive pools relative to the number of threads most deployments run. The module parameter and netlink interfaces are preserved for backward compatibility, but writing any value other than 'pernode' will return an error. The default pool mode is hardcoded to pernode, which provides one pool per node with proper thread affinity and NUMA-local memory allocation.

🔥 TODAY Re: [PATCH v4 5/8] fuse: {io-uring} Allow reduced number of ring queues Needs Work LLM (per-reviewer) Under Review

This patch proposes simplifying the logic for dispatching requests in the fuse io-uring implementation by using the group_cpus_evenly() function, which partitions CPUs based on numa nodes and other considerations. This would eliminate the need for per-numa state tracking and simplify queue selection to a simple cpu-to-qid lookup. Additionally, it suggests declaring the total queue count upfront and establishing the mapping at ring creation time, eliminating the rebuild-on-each-queue-creation complexity.
