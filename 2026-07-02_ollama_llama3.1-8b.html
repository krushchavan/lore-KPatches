This patch series adds support for io-uring buffer pools and zero-copy to the FUSE file system. The current implementation tightly couples entries and buffers, requiring a dedicated payload buffer for each entry. 🔥 TODAY [PATCH v2] iommu/arm-smmu-v3: Shrink command/event/PRI queues in kdump kernel v1 → v2 Needs Work LLM (per-reviewer) New Version Expected

This patch reduces the size of command, event, and PRI queues in the kdump kernel to a single page when running from a small crashkernel reservation. This is done by clamping each queue's maximum size based on the entry size and the log2 of the entry size. The goal is to reduce memory usage in systems with many SMMUv3 instances without affecting dump throughput, as these queues only carry invalidation commands and fault records, not DMA data.

[PATCH 7.1.y] userfaultfd: gate must_wait writability check on pte_present() Neutral llm-chunked-no-replies Awaiting Review

This patch fixes a bug in the userfaultfd code where the must_wait writability check was not properly gated on pte_present(). This caused incorrect results when checking if a fault is still write-protected after a swap or migration. The fix involves adding a check for pte_present() before applying huge_pte_write(), ensuring that only present PTE bits are inspected. This prevents the lockless re-check from returning undefined results due to concurrent migration or swap operations.

Ongoing Submitted 2026-07-01 🔥 TODAY [PATCH v2] btrfs: replace writeback inhibition xarray with a fixed inline buffer v1 → v2 Neutral LLM (per-reviewer) Under Review

This patch replaces a dynamic xarray used to track inhibited extent buffers in Btrfs with a fixed inline buffer, managed by a CLOCK eviction policy. The xarray was causing two issues: a write_iops regression due to the overhead of storing and retrieving data on every copy-on-write (COW) operation, and an unbounded cleanup walk that could lead to multi-second stalls. The new approach uses a small fixed array to track the working set of buffers revisited across COW operations, with a CLOCK eviction policy to reclaim write-once buffers first. This change is expected to improve performance by reducing the overhead of tracking inhibited buffers and preventing long-running cleanup walks.
