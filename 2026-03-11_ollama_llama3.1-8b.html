Last comment 2026-03-11 [PATCH] mm: switch deferred split shrinker to list_lru Needs Work LLM Under Review

This patch updates the deferred split shrinker to use list_lru, which properly sets up per-node and per-cgroup lists. This change improves handling of cgroups on NUMA systems by avoiding splitting large pages across nodes. The patch also streamlines list operations and reclaim walks, and adds calls to create node heads for cgroups when splittable pages are created. Additionally, it introduces helpers to the list_lru API for caller-side locking. The deferred split queue lock is replaced with a more efficient approach using list_lru's locking mechanisms.

Ongoing Submitted 2026-03-07 Last comment 2026-03-08 [PATCH v2] mm/mempolicy: track page allocations per mempolicy Needs Work LLM Under Review

This patch adds per-policy page allocation counters to track allocations driven by each memory policy (mempolicy) on a NUMA node. The new counters, exposed in /proc/vmstat and memory.numa_stat, follow the existing numa hit/miss/foreign style and provide detailed information about intentional vs fallback allocations. The counters are tracked per-memcg (memory control group) and per-node, allowing for more accurate investigation of pressure on a NUMA node.
