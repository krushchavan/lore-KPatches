This patch addresses an issue where per-node vmstat counters become stranded when a node is marked offline and then comes back online. The existing code would zero out the per-cpu counters, causing permanent skew in the statistics. This patch instead folds the stranded deltas into the global counter before the node rejoins the online set, ensuring accurate statistics. The patch has been tested with repeated memory hotplug/unplug cycles and has been found to resolve the issue.

This fix should improve performance on tiered systems with cachelib-based workloads.

Ongoing Submitted 2026-06-29 🔥 TODAY [PATCH v5 0/4] arm64: cross-CPU NMI via SDEI v4 → v5 Contentious LLM (per-reviewer) Accepted

This patch series adds a new mechanism to deliver interrupts across CPUs on arm64 systems, specifically targeting scenarios where the CPU has its interrupts locally masked. The current pseudo-NMI and FEAT_NMI mechanisms have limitations, such as introducing a significant performance overhead or requiring architectural support that is not yet widely deployed. This series introduces SDEI (System Description Event Infrastructure) as a new delivery backend, which allows firmware to deliver an interrupt into a CPU regardless of its DAIF state, thus enabling the backtrace and crash-stop paths to reach a masked CPU without incurring significant performance costs.

This patch adds a new mechanism to the Linux kernel on arm64 architecture to handle CPUs that are wedged and ignoring interrupts. When a CPU is wedged, it ignores stop IPIs and without pseudo-NMI, there's no NMI IPI to escalate to. Ongoing Submitted 2026-06-24 🔥 TODAY [PATCH] btrfs: replace writeback inhibition xarray with a fixed inline buffer Needs Work LLM (per-reviewer) Under Review

This patch replaces the xarray used to track inhibited extent buffers in Btrfs transactions with a fixed inline buffer, managed by a CLOCK eviction policy. This change addresses two issues: a write_iops regression caused by frequent xarray stores during COW operations and an unbounded cleanup walk due to long iteration over inhibited buffers. The new approach uses a small fixed-size array to track the working set of buffers revisited across search_again descents, with a CLOCK eviction policy that ensures frequently accessed buffers are retained while less frequently accessed ones are reclaimed first.
