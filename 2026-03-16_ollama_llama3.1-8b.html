Ongoing Submitted 2026-03-10 🔥 TODAY [PATCH v2 2/2] locking: Add contended_release tracepoint v1 → v2 Needs Work LLM (per-reviewer) Under Review

This patch adds a new tracepoint called contended_release, which fires when a lock is released under contention. The patch complements existing contention_begin and contention_end tracepoints by providing information from the holder's perspective. It enables correlating lock hold time under contention with waiter events by lock address. The patch introduces calls to trace_contended_release() in slowpath unlock paths of various sleepable locks, including mutex, rtmutex, semaphore, rwsem, percpu-rwsem, and RT-specific rwbase locks.

🔥 TODAY Re: [PATCHBLIZZARD v7] fuse/libfuse/e2fsprogs: containerize ext4 for safer operation Needs Work LLM (per-reviewer) Accepted

This patch proposes containerizing ext4 for safer operation, allowing for isolation and security benefits without requiring near-native performance. The approach involves using lklfuse, which is a user-space filesystem that uses the Linux kernel's virtual machine (VM) to run a kernel instance in user space. This method has pros such as easier setup and maintenance compared to implementing a fuse-iomap server for every filesystem, but may have worse performance. If near-native performance is not a requirement, using lklfuse seems like a less painful option.

Ongoing Submitted 2026-03-07 Last comment 2026-03-12 ⏱ No recent activity [PATCH v2] mm/mempolicy: track page allocations per mempolicy Needs Work LLM (per-reviewer) Under Review

This patch adds per-policy page allocation counters to track allocations driven by each memory policy (mempolicy) on a NUMA node. The new counters, exposed in /proc/vmstat and memory.numa_stat, follow the existing numa hit/miss/foreign style and provide detailed information about intentional vs fallback allocations. The counters are tracked per-memcg (memory control group) and per-node, allowing for more accurate investigation of pressure on a NUMA node.
