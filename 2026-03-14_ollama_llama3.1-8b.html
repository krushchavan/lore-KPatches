  • [3/4] elf: align ET_DYN base to exec folio order for contpte mapping 2 reviewers Submitted 2026-03-10Last comment 2026-03-14
  • [4/4] mm: align file-backed mmap to exec folio order in thp_get_unmapped_area 1 reviewer Submitted 2026-03-10Last comment 2026-03-14
Needs Work LLM (per-reviewer) Under Review

This patch series addresses issues with contpte-sized exec folios on arm64, specifically for 16K and 64K page configurations. The first issue is that the current implementation returns an incorrect order for these page sizes, which is fixed by using a different calculation. The second issue is that readahead is disabled after 100 page faults due to the mmap_miss heuristic, but this only happens on 64K pages because the decrement paths are inactive. This is fixed by moving the VM_EXEC readahead block above the mmap_miss check. Finally, the patch ensures that the virtual address is aligned to CONT_PTE_SIZE (2M) for contpte mapping, which requires alignment of the VMA start address.

Ongoing Submitted 2026-03-12 🔥 TODAY [PATCH v3] mm: migrate: requeue destination folio on deferred split queue v1 → v2 → v3 Needs Work LLM (per-reviewer) Accepted

This patch fixes a bug in the Linux kernel's folio migration code by re-queuing the destination folio on the deferred split queue after a successful migration. The source folio is removed from the queue, but the destination folio remains and can cause underutilized THPs to escape the shrinker. The patch records whether the source folio was on the deferred split queue and its partially mapped state before unqueuing it, and re-queues the destination folio if necessary. This ensures that fully mapped but potentially underused folios are correctly handled.
