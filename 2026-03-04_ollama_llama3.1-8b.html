Ongoing Submitted 2026-02-26 [PATCH v4 1/3] btrfs: skip COW for written extent buffers allocated in current transaction v3 → v4 Needs Work LLM Under Review

This patch optimizes Btrfs by skipping unnecessary copy-on-write (COW) operations when a buffer has already been written in the current transaction. When memory pressure causes writeback of a recently COW'd buffer, Btrfs sets a flag indicating it's been written. Subsequent searches then re-COW the buffer unnecessarily, leading to COW amplification and potential performance degradation. The patch addresses this issue by re-dirtying the buffer and re-registering its range in the transaction's dirty_pages when the generation matches the current transaction. This change ensures that overwriting in place is crash-safe and prevents unnecessary COW operations.

Ongoing Submitted 2026-02-26 [PATCH v4 2/3] btrfs: inhibit extent buffer writeback to prevent COW amplification v3 → v4 Needs Work LLM Under Review

This patch prevents COW amplification in Btrfs by inhibiting background writeback of extent buffers while a transaction handle is still modifying them. It introduces an atomic counter to track the number of inhibited buffers and uses this count to skip writeback when necessary. The patch also ensures that all handles accessing a COW'd buffer contribute to the inhibitor count, preventing the buffer from being unprotected prematurely. This approach avoids unnecessary overhead by preventing re-COW operations when a buffer is still being modified.
