This patch converts the task_struct's objcg field to a per-node array of obj_cgroup pointers, allowing for efficient and accurate accounting of memory usage across multiple NUMA nodes. The change addresses issues with current_obj_cgroup() returning an objcg whose nid did not match the current CPU's node, defeating per-node vmstat batching and accounting hierarchy. [PATCH] btrfs: add EM_WARN_ON_ONCE() to dump the extent map on a warning Neutral llm-chunked-no-replies Awaiting Review

This patch adds a new macro EM_WARN_ON_ONCE() to the Btrfs file system, which dumps the contents of an extent map to the kernel log before triggering a warning and stack trace when certain consistency checks fail. The patch addresses a problem where WARN_ON() warnings were not providing enough information about the offending extent maps, leading to crashes in production kernels. The EM_WARN_ON_ONCE() macro is always compiled in, unlike other similar macros that are gated behind configuration options, ensuring that it can provide actionable context even on production systems.

Ongoing Submitted 2026-06-25 🔥 TODAY [PATCH v2] sunrpc: hardcode pool_mode to pernode, remove other modes Needs Work LLM (per-reviewer) Under Review

This patch removes the SVC_POOL_* enum and related code, hardcoding the pool mode to pernode (one pool per NUMA node) as it is considered the best choice today. The percpu mode, which created excessive pools, is removed along with its auto-selection logic. The module parameter and netlink interfaces are preserved for backward compatibility. If pool map allocation fails, service creation now fails instead of silently falling back to a single global pool.

This patch series converts the iomap code to use an in-iter model, where a single callback function is used for both beginning and ending I/O operations. The conversion involves merging two existing functions into one and adding a helper function to process I/O operations. An example of devirtualizing the callbacks on a hot path is also provided to avoid indirect calls through struct iomap_ops. The patch series raises questions about whether this conversion is worth it, given that indirect calls are cheap on modern hardware.

Needs Work LLM (per-reviewer) Under Review

This patch series moves the per-cpu stock from struct mem_cgroup to page_counter, reducing memory footprint and eliminating locking and synchronous work_on_cpu. The new implementation uses atomic operations for stock management and allows for independent stock management for additional page_counters in each memcg. This change simplifies memcontrol code and enables the addition of a new page_counter to track toptier usage. However, it introduces a potential increase in memory overcharging on large machines with many cgroups.
