Needs Work LLM (per-reviewer) Under Review

This patch set improves write performance when using the RWF_DONTCACHE flag by kicking the flusher thread for the superblock instead of initiating writeback directly. This change is based on benchmarking results that show a clear improvement over the status quo, particularly in cases with heavy write activity. The patch also includes a comparison of different IO modes for nfsd and highlights the poor performance of RWF_DONTCACHE with write-heavy workloads. The approach taken involves modifying the generic_write_sync function to kick the flusher thread, which is expected to better isolate the DONTCACHE writeback behavior and improve overall performance.

This patch adds support to CB_NOTIFY in the NFS server to send notifications when directory attributes change, in addition to add/remove/rename events. The server will still recall the delegation on a SETATTR operation, so these notifications are only sent for changes to child directories. The patch modifies two files: nfsd/nfs4state.c and fs/nfsd/nfs4xdr.c, adding new code to handle directory attribute change notifications and update the notify entry structure accordingly.

This patch adds a test to the server41tests suite in the Linux kernel's NFSv4 implementation. The test, called 'testDirDelegRenameOver', verifies that when a within-directory rename operation overwrites an existing file entry, the server correctly populates the nad_old_entry field in the NOTIFY4_RENAME_ENTRY notification with the overwritten entry's information. This is in accordance with RFC 8881bis Section 27.4.6. The test creates two files in a delegated directory, renames one of them over the other, and then checks that the server sends a correct RENAME notification with the old entry's information.
