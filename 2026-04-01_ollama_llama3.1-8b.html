Last comment 2026-04-02 Re: [PATCH 0/6] btrfs: delay compression to bbio submission time Needs Work LLM (per-reviewer) Changes Requested

This patch series delays compression and allocation until bbio submission time, allowing for earlier unlocking of delalloc ranges during writeback. This is achieved by creating a delayed ordered extent (OE) that provides sufficient synchronization to break the contract between OE and allocated space. The new approach involves submitting delayed bios in the writeback context, which are then processed asynchronously. The patch series also introduces a more natural model for handling ranges like OEs through splitting/joining instead of layering into a tree. This change aims to reconcile the iteration in writeback with delalloc locking and iteration, making it easier to manage extents correctly.

🔥 TODAY [PATCH v1] fuse: fix io-uring background queue stall on request completion Needs Work LLM (per-reviewer) Under Review

This patch fixes a stall in the io-uring background queue when a request completes. The issue arises from the fact that the connection-level background counters are not properly accounted for before flushing the queue's background queue, leaving pending requests stuck. To fix this, the patch accounts for the connection-level counters first and then flushes the queue's background queue, ensuring that pending requests are dispatched correctly.

Needs Work LLM (per-reviewer) Under Review

This patch series switches the THP (Transparent Huge Pages) shrinker to use list_lru instead of an open-coded deferred split queue. The current implementation has issues, including being non-NUMA-aware and complicated in its interactions with other code. Using list_lru fixes these problems and simplifies planned shrinker work. The patch series includes several cleanups and refactors in the list_lru code to make the conversion easier, as well as extensions to the list_lru API to allow for more control over locking scope and a convenience helper for allocating LRU heads when coming from a folio.

This patch series addresses two technical inaccuracies in the TDX guest port I/O emulation code, specifically an off-by-one error in GENMASK() macro usage and incorrect zero-extension for 32-bit port I/O operations. The patches fix these issues by correcting the mask calculation and ensuring that the result is correctly extended to a full 64-bit RAX register. These changes were introduced in the initial implementation of runtime hypercalls for port I/O and are necessary to maintain accurate emulation of x86 architecture requirements.
