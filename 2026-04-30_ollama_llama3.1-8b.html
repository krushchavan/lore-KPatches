This patch addresses a potential lock imbalance issue in the VFS directory operations by ensuring that the parent lock is reacquired before exiting early from certain lookup functions. The problem arises when d_alloc_parallel() or d_alloc() return without holding the lock, causing the caller to unlock an unheld rwsem and corrupting its state. 🔥 TODAY Re: fuse/io-uring: Proposal to support pBuf in additon to kBuf Needs Work LLM (per-reviewer) Accepted

This patch proposes to support pBuf in addition to kBuf in the fuse/io-uring interface, allowing for more efficient handling of small requests and reducing kernel-side complexity. The current implementation using multiple queues for different buffer sizes is seen as suboptimal due to fragmentation and inefficient distribution of buffers. The proposed solution involves separate payload-size queues, which simplifies kernel-side logic and improves performance. This change aims to address issues with head-of-line blocking and non-blocking IO, making it more feasible for fuse servers to use io-uring without requiring significant modifications.

Needs Work LLM (per-reviewer) Superseded

This patch series adds support for tracking the working set of VM guest memory using userfaultfd (UFFD). It introduces a new registration mode, UFFDIO_REGISTER_MODE_RWP, which allows a VMM to identify cold pages and evict them to slower storage. The kernel uses a combination of PTE bits and uffd PTE bit to mark pages as 'userfaultfd-tracked' and keep them resident while inaccessible. In sync mode, the kernel delivers a UFFD_PAGEFAULT_FLAG_RWP message to the registered handler, while in async mode, the fault is auto-resolved in-place. Userspace can then learn which pages were touched by reading PAGE_IS_ACCESSED out of PAGEMAP_SCAN. The series also includes preparatory patches to decouple protnone helpers from CONFIG_NUMA_BALANCING and rename _PAGE_BIT_UFFD_WP-related functions.

Neutral llm-chunked-no-replies RFC

This patch replaces the packed pageblock_flags bitmap in the Linux kernel's memory management code with a per-pageblock struct called pageblock_data, which contains its own flags word. This change simplifies the accessor code and reduces cache misses by colocating migratetype and pageblock ownership lookups in the same struct. The memory overhead of this change is 8 bytes per pageblock, but it is considered negligible. No functional changes are made.
