Ongoing Submitted 2026-07-09 🔥 TODAY [PATCH] btrfs: write-protect folios during data writeback Needs Work LLM (per-reviewer) Under Review

This patch addresses a problem where, due to a recent change in the Btrfs kernel module, folios (large blocks of data) are not being properly write-protected during data writeback operations. This can lead to corruption and bugs when processes try to modify sectors while they're being used by writeback. The patch reintroduces the necessary write-protection mechanism without dropping the dirty flag on the folio, which would cause other issues. It also adds a debug warning to help prevent similar regressions in the future.

Needs Work LLM (per-reviewer) Under Review

This patch series adds atomic whole-device hotplug control via sysfs, allowing the state of an entire region to be controlled in a single operation. The current implementation onlines memory during probe using the system default policy, but this can lead to races between offlining and removing a whole region. The new feature provides a 'state' attribute that mirrors the per-block memoryX/state ABI, with states for unplugged, online, online_kernel, and online_movable. This allows for atomic transitions across every range of the device, solving the problem of interfering userland controllers during offline and remove steps.
