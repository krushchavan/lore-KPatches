[PATCH v2 1/3] mm: swap: overlap synchronous readahead decompression with disk I/O 3 patches Neutral llm-chunked-no-replies Awaiting Review

This patch optimizes the swap readahead process by overlapping synchronous decompression of zswap-resident entries with disk I/O operations. The current implementation serializes CPU work in front of disk reads, introducing unnecessary latency. To address this, the patch splits the window walk into two passes: one issues asynchronous device reads and defers synchronous neighbors, while the second pass reads deferred entries concurrently with ongoing I/O operations. This approach reduces added latency and improves swap-in performance.

[PATCH v2 3/3] mm: swap: treat SWP_SYNCHRONOUS_IO devices as synchronous in readahead
Show 1 individual patches
  • [2/1] mm: swap: bypass synchronous readahead unless the target needs I/O
Neutral llm-chunked-no-replies Awaiting Review

This patch modifies the Linux kernel's swap handling to treat devices marked as SWP_SYNCHRONOUS_IO (such as zram) as synchronous in readahead operations. This change allows for better overlap of I/O operations between disk and zram, improving performance in mixed workloads. The modification extends the swap_entry_synchronous() function to take into account the swap device and returns true for entries on SWP_SYNCHRONOUS_IO devices, ensuring that these reads are deferred until the second pass. This patch is part of a series aimed at optimizing swap handling and has been tested with positive results in mixed zram+disk workloads.

This patch improves the performance of zswap by freeing cold writeback folios promptly after writeback completion, reducing unnecessary reclaim scans and keeping memory resident for shorter periods. The patch consists of two parts: one for synchronous-IO devices (zram, brd, pmem) that can free the folio directly in the calling context, and another for asynchronous block devices and filesystem-backed swap that allocate the folio off the LRU and defer the free to a workqueue. The results show significant improvements in performance metrics such as page rotations, steals, scans, and build times, with most folios being freed promptly on writeback completion.
