This patch adds a new netlink command, NFSD_CMD_CACHE_FLUSH, which allows userspace to flush the nfsd export caches without writing to /proc/net/rpc/*/flush. The command takes an optional attribute, NFSD_A_CACHE_FLUSH_MASK, that selects which caches to flush. If omitted, all nfsd caches are flushed. This is used by exportfs to replace its /proc-based cache_flush() with a netlink equivalent, with /proc fallback for older kernels.

Ongoing Submitted 2026-03-18 🔥 TODAY [PATCH v1] writeback: skip sync(2) inode writeback for filesystems with no data integrity guarantees Neutral LLM (per-reviewer)

This patch adds a new superblock flag, SB_I_NO_DATA_INTEGRITY, to identify filesystems that cannot guarantee data persistence on sync operations. It then uses this flag to skip the inode writeback for these filesystems during sync(2) operations, improving efficiency and preventing potential hangs. The change replaces a per-inode flag with a superblock-level flag, as data integrity is a filesystem-wide property. This patch restores fuse's previous behavior before tmp folios were removed, where sync was essentially a no-op.

This patch series switches the THP (Transparent Huge Pages) shrinker to use list_lru instead of an open-coded deferred split queue. The current implementation has issues, including being non-NUMA-aware and complicated in its interactions with other code. Using list_lru fixes these problems and simplifies planned shrinker work. Neutral LLM (per-reviewer)

This patch series addresses issues with contpte-sized exec folios on arm64, specifically for 16K and 64K page configurations. The first issue is that the current implementation returns an incorrect order for these page sizes, which is fixed by using a different calculation. The second issue is that readahead is disabled after 100 page faults due to the mmap_miss heuristic, but this only happens on 64K pages because the decrement paths are inactive. This is fixed by moving the VM_EXEC readahead block above the mmap_miss check. Finally, the patch ensures that the virtual address is aligned to CONT_PTE_SIZE (2M) for contpte mapping, which requires alignment of the VMA start address.
