[PATCH v2] fuse: end fuse_req on io-uring cancel task work Positive llm-chunked-no-replies Awaiting Review

This patch fixes a bug in the io-uring implementation for FUSE (Filesystem in Userspace) where a fuse request is not properly ended when an io-uring task is cancelled. When an io-uring task is cancelled, the fuse request remains linked to the processing queue and its reference count is never decremented, leading to resource leaks and potential crashes. The patch fixes this by releasing the entry directly from the cancel branch, removing it from the queue, completing the io-uring command, ending the fuse request, freeing the entry, and dropping its queue references.

[PATCH v2 0/3] arm64: cross-CPU NMI via SDEI v1 → v2 Positive llm-chunked-no-replies Accepted

This patch series adds support for cross-CPU NMI (Non-Maskable Interrupt) on arm64 using the SDEI (System Description Event Infrastructure) mechanism. This is necessary because current mechanisms, such as pseudo-NMI and FEAT_NMI, have significant performance costs or are not available in deployed silicon. The new SDEI-based approach allows for interrupting a CPU with its interrupts locally masked without impacting performance, enabling more accurate debugging and observability features like backtraces and crash dumps.
