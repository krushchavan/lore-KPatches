This patch converts the nvdimm_bus guard from using DEFINE_GUARD() to a combination of DEFINE_CLASS() and DEFINE_CLASS_IS_GUARD(). This change is necessary because the upcoming removal of NULL checks in DEFINE_GUARD() constructors would cause a compile error for nvdimm_bus, which can pass NULL. Ongoing Submitted 2026-05-21 ⏱ No comments yet [PATCH v4 4/4] cleanup: Remove NULL check from unconditional guards v2 → v3 → v4 Neutral llm-chunked-no-replies Awaiting Review

This patch removes unnecessary NULL checks from unconditional guards in the Linux kernel. The checks were dead code because the guards guarantee a non-NULL lock pointer at destructor time. The patch affects several macros, including DEFINE_GUARD(), DEFINE_LOCK_GUARD_1(), and DEFINE_LOCK_GUARD_0(). It also fixes a few specific instances where the NULL check was not being used correctly. The changes are verified to have no performance impact, with a negligible reduction in code size.

This patch series adds buffer pools and zero-copy functionality to the fuse io-uring implementation, allowing servers to pass in a managed buffer pool that reduces memory usage requirements and enables future optimizations. It also allows for direct access to client pages or page cache folios without copying data through an intermediary buffer, requiring CAP_SYS_ADMIN privileges. The changes are based on registered buffers and zero-copy, which can provide significant throughput improvements depending on the server's per-request latency and backing I/O characteristics.
