Ongoing Submitted 2026-03-06 🔥 TODAY [PATCH 1/8] mm: use zone lock guard in reserve_highatomic_pageblock() Needs Work LLM (per-reviewer) Accepted

This patch updates the reserve_highatomic_pageblock() function in the Linux kernel's page allocation code to use a zone lock guard instead of explicit lock/unlock and goto out_unlock patterns. The zone lock guard is a new feature introduced by this patch, which provides automatic scope-based cleanup when exiting the locked region. This change simplifies the code and reduces the risk of errors due to forgotten unlocks or incorrect locking orders. The patch also updates the include/linux/mmzone_lock.h file to define the zone lock guard and its attributes.
