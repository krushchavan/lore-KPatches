This patch modifies the Linux kernel's page allocation mechanism to allow high-priority non-blocking allocations to use existing MIGRATE_HIGHATOMIC reserves on the buddy fastpath, rather than falling back to other migratetypes. The ALLOC_HIGHATOMIC flag is updated to only provide access to these reserves without permission to grow them, and a new flag ALLOC_HIGHATOMIC_RESERVE is introduced to specifically allow growing the reserves. This change improves performance in certain workloads by reducing the number of highatomic misses.

Ongoing Submitted 2026-06-17 🔥 TODAY [PATCH v4 0/4] arm64: cross-CPU NMI via SDEI v2 → v3 → v4 Needs Work LLM (per-reviewer) Under Review

This patch series adds a new mechanism to deliver NMI (Non-Maskable Interrupt) across CPUs on arm64 architecture using SDEI (System Description Event Infrastructure). This is necessary for certain debug and observability features, such as capturing CPU backtraces and crash dumps, which require interrupting a CPU that has its interrupts locally masked. The existing pseudo-NMI mechanism is too expensive in terms of performance overhead, so this new approach uses SDEI to deliver the NMI at a lower cost. This allows for more accurate debugging and observability features without introducing significant performance regressions.

Ongoing Submitted 2026-06-17 Last comment 2026-06-17 ⏱ No recent activity [PATCH v4 4/4] arm64: escalate smp_send_stop() to an SDEI NMI as a last resort v2 → v3 → v4 Needs Work LLM (per-reviewer) Under Review

This patch adds a new mechanism to the Linux kernel on arm64 architecture to handle CPU wedging by escalating smp_send_stop() to an SDEI NMI (Non-Maskable Interrupt) as a last resort. When a CPU is wedged with interrupts masked, it ignores stop IPIs and without pseudo-NMI there's no NMI IPI to escalate to. The patch introduces a third rung in smp_send_stop() that signals an SDEI event 0 at the remaining online CPUs, which firmware delivers regardless of the target's DAIF (Debug Access Interface Flag). This allows the CPU to reach a wedged CPU and acknowledge by going offline. The stop bookkeeping is also folded into a shared function arm64_nmi_cpu_stop() for consistency.
