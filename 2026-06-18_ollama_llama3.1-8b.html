This patch series fixes a regression in perf record's Zstandard compression feature, which caused it to abort recording and produce an undecompressable file due to a 'Bad address' error. [PATCH 2/3] perf record: Avoid overrunning the zstd output buffer Neutral llm-chunked-no-replies Awaiting Review

This patch fixes a bug in the zstd compression API used by perf tools, where the output buffer could be overrun when compressing data. The issue occurred because the function did not check if there was enough space left in the output buffer to hold a full record before writing one. The patch adds a check to prevent this overwriting and returns an error instead.

[PATCH 3/3] perf record: Don't store raw data on zstd compression failure Neutral llm-chunked-no-replies Awaiting Review

This patch fixes a bug in the perf record tool's ZSTD compression handling. When ZSTD compression fails, it previously attempted to copy the uncompressed data into the output buffer, which can lead to issues such as incorrect header sizes and unbounded copying. Instead, this patch propagates the error by returning -1 when compression fails, preventing these problems.

[PATCH nfs-utils] nfsstat: display NFSv4 callback operation statistics Neutral llm-chunked-no-replies Awaiting Review

This patch adds a new feature to the nfsstat utility, which displays NFSv4 callback operation statistics. The kernel maintains per-operation call counts for NFSv4 backchannel operations and exports them through the server-stats netlink dump. The patch updates the nfsstat code to report these counts in a 'Server nfs v4 callback operations' section when the stats are fetched via netlink. This feature is only available over netlink, as the /proc interface does not expose equivalent data.
