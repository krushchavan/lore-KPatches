Ongoing Submitted 2026-06-23 🔥 TODAY [PATCH 0/5] allocate extent_buffer GFP_NOFAIL with unlocked retry Needs Work LLM (per-reviewer) Under Review

This patch series addresses contention and stalls in the Linux kernel caused by btrfs extent_buffer allocations being made while holding locks. The patches modify the allocation of extent_buffers to use GFP_NOWAIT, which allows the allocation to fail and retry outside of the critical section with GFP_NOFS | GFP_NOFAIL. This approach is analogous to how extent_buffers are read, where locks must be dropped before accessing the data. Although the series does not completely eliminate lock-holding allocations, it significantly reduces contention by allowing small cached slab allocations to occur outside of the critical path.

Ongoing Submitted 2026-06-23 🔥 TODAY [PATCH 1/5] btrfs: factor init_extent_buffer from __alloc_extent_buffer Needs Work LLM (per-reviewer) Under Review

This patch refactors the initialization of extent buffers in Btrfs by separating it from the allocation process. The goal is to prepare for preallocating extent buffer data, which will allow for more efficient memory management. The patch introduces a new function `init_extent_buffer` that initializes an existing extent buffer, and modifies the existing `__alloc_extent_buffer` function to allocate the buffer first and then initialize it using the new function. This change enables the allocation of buffers in multiple code paths while still sharing initialization code.

This patch adds a new structure called `struct btrfs_eb_prealloc` to carry the extent_buffer and btrfs_folio_state needed for NOFAIL allocations with retries outside the critical section. The existing allocation pathways are refactored to use this new struct, but no functional changes are made. Empty prealloc structs are wired in from callers. This is part of a larger series of patches to support NOFAIL allocations.

Ongoing Submitted 2026-06-23 ⏱ No comments yet [PATCH 3/5] btrfs: enable unlocked NOFAIL retry for eb allocations Neutral llm-chunked-no-replies Awaiting Review

This patch enables unlocked NOFAIL retry for eb allocations in the Btrfs file system. It introduces a new preallocation struct to carry allocation and signal requirements, allowing callers to attempt NOWAIT allocations if supported by the allocator. If the initial allocation fails, it signals that preallocation is required, enabling the caller to perform the necessary allocation and retry with the preallocated resources. The patch also notes that two GFP_NOFS allocations remain under lock and cannot be preallocated due to limitations in the filemap xarray.
