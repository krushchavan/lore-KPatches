This patch introduces a new concept of 'delayed ordered extent' in Btrfs, which is a range of file extents without submitted I/O or space allocation. The existing code for finishing one ordered extent can be reused on the children of this delayed OE, with only minor modifications. The idea is to split the concepts of ordered extent and its parent into two separate entities: one representing the range of file extents in the rb-tree, and the other representing the I/O operations. This change simplifies the code and reduces the need for special handling of parent-child relationships between OEs.

[PATCH v2] mm/page_alloc: fix defrag_mode for non-reclaimable allocations v1 → v2 Neutral llm-chunked-no-replies Awaiting Review

This patch fixes a problem in the Linux kernel's page allocation system where non-reclaimable allocations, such as those made with GFP_ATOMIC, fail when defrag_mode is enabled. The issue arises because these allocations cannot invoke direct reclaim or compaction, which are necessary for defrag_mode to work correctly. To fix this, the patch clears the ALLOC_NOFRAGMENT flag and retries the allocation if it requests kswapd reclaim but cannot do direct reclaim itself. This change allows non-reclaimable allocations to fall back to other migration types when defrag_mode is enabled, preventing a large number of SLUB allocation failures under network-heavy workloads.

🔥 TODAY [PATCH] sunrpc: use kref_get_unless_zero in auth_domain_lookup Needs Work LLM (per-reviewer) Under Review

This patch fixes a bug in the auth_domain_lookup function of the sunrpc module by using kref_get_unless_zero instead of plain kref_get to acquire a reference to an auth_domain entry. This prevents a window where an entry is still linked on the hash list with refcount == 0, which could lead to a dangling reference and object freeing while the caller still holds a pointer to it. The patch applies the same pattern used in the sibling function auth_domain_find to preserve existing semantics.
