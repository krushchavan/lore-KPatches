This patch fixes a problem where the defrag_mode in the Linux kernel's page allocator fails to handle non-reclaimable allocations, such as those made with GFP_ATOMIC flags. When defrag_mode is enabled, it enforces ALLOC_NOFRAGMENT to prevent fragmentation, but this causes failures for non-reclaimable allocations that cannot invoke direct reclaim or compaction. The patch clears the ALLOC_NOFRAGMENT flag and retries the allocation if it requests kswapd reclaim but cannot do direct reclaim itself. This fix addresses a large number of SLUB allocation failures under network-heavy workloads.

🔥 TODAY Re: [PATCH] fuse: disable default bdi strictlimiting Needs Work LLM (per-reviewer) Under Review

This patch disables default strictlimiting in the fuse (Filesystem in Userspace) module to prevent performance regressions caused by excessive write throttling due to large folio sizes and limited dirty budgets. The current implementation can lead to severe performance issues, including oscillating writes between freerunning and full throttling, especially on systems with limited RAM or cgroups with memory constraints. By disabling strictlimiting, the patch aims to prevent misbehaving servers from accumulating too many dirty pages, which can degrade write throughput for other filesystems.

This patch modifies the Btrfs file system to prevent finalizing file system roots during qgroup snapshot accounting. This approach ensures that the transaction remains in progress and allows for future optimizations to rely on the FORCE_COW flag correctly.

[PATCH v2 2/2] btrfs: skip COW for written extent buffers allocated in current transaction Neutral llm-chunked-no-replies Awaiting Review

This patch optimizes Btrfs by skipping unnecessary copy-on-write (COW) operations when a buffer is written in the current transaction. When memory pressure causes writeback of a recently COW'd buffer, Btrfs sets a flag indicating it has been written. Subsequent searches then re-COW the buffer unnecessarily, leading to COW amplification and potential performance degradation. The patch addresses this issue by re-dirtying the buffer and re-registering its range in the transaction's dirty_pages when the generation matches the current transaction. This approach ensures that the buffer is not unnecessarily re-written, while maintaining crash safety and proper handling of dirty metadata.

Ongoing Submitted 2026-05-26 ⏱ No comments yet [PATCH v2] btrfs: allocate eb-attached btree pages as movable Neutral LLM (per-reviewer) Under Review

This patch modifies the allocation of btree pages in Btrfs to make them movable, which allows the page allocator to move them around and reduce fragmentation. The change involves passing a full GFP mask directly to functions that allocate pages, rather than having them calculate it from boolean flags. This is necessary because the current behavior causes btree_inode pages to be pinned as unmovable, even though they can be moved. The patch targets a common problem of fragmentation in Btrfs, where ~28 GB of btree_inode pages are parked across many tainted superpageblocks, preventing large allocations. With this change, those pages now land in movable pageblocks and can be drained by the background defragger.

This patch series aims to improve the allocation of large memory blocks by dividing memory into superpageblocks and steering unmovable, reclaimable, and high-atomic allocations into those already tainted by such allocations. The goal is to leave as many 1GB superpageblocks available for movable allocations, which can be easily defragmented for regular or huge pages. Various strategies are employed to achieve this, including preferential allocation from tainted blocks, fallback to small pages for certain kernel allocations, and migration of movable memory when a block falls below a free threshold. This approach enables more extensive use of 2MB THPs and mTHPs, as well as reliable allocation of 1GB pages. The series is built on top of Johannes's PCPBuddy series, which aims to reduce the need for CPUs to get pages from zone free lists by relying on page block ownership.
