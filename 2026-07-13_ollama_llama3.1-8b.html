  • [1/2] mm/filemap: place page-cache folios via an explicit mempolicy Submitted 2026-07-01
  • [2/2] mm/mempolicy: skip task mempolicy for unmovable unaccounted kernel allocations Submitted 2026-07-01
Needs Work LLM (per-reviewer) Under Review

This patch series prevents a task's NUMA memory policy from influencing incidental kernel allocations that have no direct relationship to the task. Currently, these allocations follow the task's policy, which can lead to poor placement on tiered-memory systems and cause regressions. The fix is to only apply the task policy to movable allocations explicitly tied to the task via __GFP_ACCOUNT, while preferring node-local placement for unmovable kernel allocations. This change ensures that kernel memory is placed in a way that minimizes performance regressions and maintains existing behavior for page cache placement.

Needs Work LLM (per-reviewer) Accepted

This patch series adds support for io-uring buffer pools and zero-copy to the FUSE file system. The current implementation tightly couples entries and buffers, requiring a dedicated payload buffer for each entry. This patch decouples them by allowing servers to pass in a buffer pool that the kernel can use as needed. Additionally, it introduces zero-copy functionality, which allows servers to directly access client pages or page cache folios without copying data through an intermediary buffer. The benefits of this patch include reduced memory usage requirements and improved throughput, particularly when backing I/O is not the bottleneck.

Last comment 2026-07-14 [PATCH] mm: thp: pin the inode across a file folio split Needs Work LLM (per-reviewer) Under Review

This patch fixes a use-after-free bug in the THP (Transparent Huge Pages) split function by pinning the inode across a file folio split using igrab(). The problem occurs when a concurrent final iput() can evict and RCU-free the inode before __folio_split() is done touching the mapping. The patch introduces a new variable 'inode' to hold the inode reference, which is grabbed with igrab() before the split and dropped with iput() after the last mapping dereference. If the inode is already being evicted, the function returns -EBUSY.

🔥 TODAY Re: [PATCH mm-hotfixes v2 0/4] mm: fix UAF caused by race between ptdump and vmap pgtable freeing Needs Work LLM (per-reviewer) Under Review

This patch series addresses an out-of-bounds memory access (UAF) caused by a race between the ptdump and vmap pgtable freeing mechanisms in the Linux kernel. The issue arises from inverted locking, where the walker takes the write lock and mutators take read locks, which is subtle and prone to deadlocks. To fix this, the patch proposes deferring the free of page tables using RCU (Read-Copy Update) and making ptdump a lockless walker. This approach would eliminate the need for inverted locking and prevent potential deadlocks, while also improving performance by reducing the load on the init_mm mmap write lock.
