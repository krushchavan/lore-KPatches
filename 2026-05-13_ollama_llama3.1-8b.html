Ongoing Submitted 2026-05-12 ⏱ No comments yet [PATCH 2/4] btrfs: fix squota accounting during enable generation Neutral llm-chunked-no-replies Awaiting Review

This patch fixes a problem in Btrfs where quota accounting is incorrect when enabling quotas for the first time. When the first transaction that enables quotas completes, it sets a flag to indicate that quotas are enabled, but this can cause a deadlock and lead to incorrect quota values. The patch introduces two changes: first, it sets the enable generation to the next transaction's ID instead of the current one, ensuring that extents written during the enabling transaction are not accounted for; second, it adds a new flag to record squota deltas for extents written between the end of the enabling transaction and when the quotas-enabled flag is set. This fixes an underflow issue where metadata tree blocks would be freed prematurely, causing mismatched usage errors in btrfs check.
