[PATCH v2 0/9] io_uring: add kernel-managed buffer rings 9 patches Needs Work LLM Under Review

This patch series introduces kernel-managed buffer rings for io_uring, where the kernel allocates and manages buffers on behalf of the application. This simplifies the process for applications by eliminating the need to allocate and manage backing buffers. The patches add support for kernel-managed buffer rings in various parts of the io_uring code, including memory mapping, buffer selection, pinning/unpinning, recycling, and command handling.

Ongoing Submitted 2026-02-12 [PATCH 0/2] improve per-node allocation and reclaim visibility 2 patches Needs Work LLM Under Review

This patch series improves visibility into per-node memory allocation and reclaim by adding new node statistics to track allocations on a per-policy basis, and moving pgscan and pgsteal counters from global vm_event_item to per-node node_stat_item. This allows for easier diagnosis of NUMA imbalance scenarios where reclaim kicks in despite available free memory.
