Ongoing Submitted 2026-04-04 🔥 TODAY [PATCH v2] mm/percpu, memcontrol: Per-memcg-lruvec percpu accounting Needs Work LLM (per-reviewer) Under Review

This patch modifies the Linux kernel's memory accounting system to provide more detailed information about per-memcg-lruvec (per-cgroup, per-node, and per-LRU vector) percpu allocations. It does this by converting a certain type of memory tracking from a per-memcg level to a per-node level, allowing for better visibility into memory pressure on multi-NUMA systems. The patch also introduces separate accounting for node-level statistics and memcg-lruvec statistics, which requires exposing a new function to be used outside of the memory control module. This change improves the kernel's ability to track memory usage at a finer granularity, particularly for percpu allocations, but has a small memory overhead and negligible performance impact.

🔥 TODAY [PATCH v4] btrfs: add BTRFS_IOC_GET_CSUMS ioctl v2 → v3 → v4 Needs Work LLM (per-reviewer) Under Review

This patch adds a new unprivileged ioctl, BTRFS_IOC_GET_CSUMS, which allows userspace to query the on-disk checksums for a file without exposing raw csum tree lookups. The ioctl takes a struct btrfs_ioctl_get_csums_args detailing the offset and length of interest, and writes a struct btrfs_ioctl_get_csums_entry into a provided buffer followed by the checksums if available. This is designed to speed up mkfs.btrfs --rootdir operations, particularly when using the --reflink option, by avoiding recalculation of checksums or reading data entirely.
