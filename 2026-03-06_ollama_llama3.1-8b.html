The implementation uses a lightweight inline helper to check if tracing is enabled, and calls an out-of-line helper when tracing is active. Positive LLM Under Review

This patch changes the type of inode->i_ino from unsigned long to u64 to prevent silent truncation of 64-bit inode numbers on 32-bit architectures. This is necessary because several filesystems can generate inode numbers that exceed 32 bits, leading to collisions and bugs on 32-bit systems. The change updates format specifiers and local variable types throughout the kernel to match the new type. This patch is part of a larger series that converts various data structures to use u64 instead of unsigned long.
