Ongoing Submitted 2026-02-27 [PATCH v4 0/5] mm: zone lock tracepoint instrumentation v2 → v3 → v4 Needs Work LLM Under Review

Ongoing Submitted 2026-03-04 [PATCH v3 12/12] treewide: change inode->i_ino from unsigned long to u64 Needs Work LLM Under Review

This patch changes the type of inode->i_ino from unsigned long to u64 to prevent silent truncation of 64-bit inode numbers on 32-bit architectures. This is done to avoid inode number collisions and other subtle bugs that can occur when filesystems generate inode numbers larger than 32 bits. The change affects various filesystems, including NFS, XFS, BTRFS, and others, and updates format specifiers and local variable types accordingly.
