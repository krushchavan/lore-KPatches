  • [4/4] sunrpc: remove unused svc_version vs_count field
Neutral llm-chunked-no-replies Awaiting Review

This patch series converts the nfsstat server-side interfaces to use netlink, allowing for per-network-namespace procedure call counts and exposing them through a new netlink handler in the nfsd generic netlink family. The procfs interface is fragile and has limitations, such as being global and not namespace-aware. This change adds per-netns counters to struct svc_stat and implements a server-stats-get netlink dump handler. It also removes unused global arrays and updates various files to use the new interfaces. The goal is to provide a more robust and extensible way for nfsstat to retrieve server statistics.

[PATCH nfs-utils] nfsstat: add netlink support for fetching server statistics Neutral llm-chunked-no-replies Awaiting Review

This patch adds support for fetching NFS server statistics using Generic Netlink, a kernel feature that allows user-space applications to communicate with the kernel. When netlink is available, nfsstat sends a dump command and parses the reply to populate various stat arrays. If netlink is unavailable (e.g., on older kernels), it falls back to parsing /proc/net/rpc/nfsd as before. The patch requires corresponding kernel patches to add netlink support.

Neutral llm-chunked-no-replies Awaiting Review

This patch optimizes the memsw stock for cgroup v1 by introducing an independent stock for the memsw page counter, which is charged separately from the memory page counter. This change improves caching and reduces the worst-case scenario where both counters exceed their limits, requiring a hierarchical charge. The user-visible effect is that reported memsw usage may temporarily be lower than memory usage due to batching of stock charges.

Ongoing Submitted 2026-05-19 ⏱ No comments yet [PATCH] mm/compaction: cap compact_gap() at COMPACT_CLUSTER_MAX Needs Work LLM Under Review

This patch caps the `compact_gap()` function at `COMPACT_CLUSTER_MAX` to prevent over-reserving memory and improve compaction success rates. The current implementation returns a gap that scales exponentially by order, but this can lead to unnecessary reclaim attempts and failed compaction checks. By capping the gap, the patch aims to align it with the actual capacity of the compaction scanner.

Ongoing Submitted 2026-05-19 🔥 TODAY [PATCH] mm/page_alloc: skip high atomic reservation at or below costly order Needs Work LLM (per-reviewer) Under Review

This patch modifies the Linux kernel's page allocation mechanism to prevent high atomic reservations for allocations at or below a certain order (PAGE_ALLOC_COSTLY_ORDER). This is done by adding a check in the reserve_highatomic_pageblock function, which skips reserving entire pageblocks for smaller atomic allocations. The goal of this change is to reduce fragmentation and improve the success rate of large memory allocations, particularly for Transparent Huge Pages (THP) on systems with plenty of free memory.
