Ongoing Submitted 2026-05-20 🔥 TODAY [PATCH v2] mm/page_alloc: fix defrag_mode for non-reclaimable allocations v1 → v2 Neutral LLM Under Review

This patch addresses a fragmentation issue in the Linux kernel's page allocator when defrag_mode is enabled and non-reclaimable allocations (e.g., GFP_ATOMIC) are made. The problem arises because these allocations cannot invoke direct reclaim or compaction, leading to SLUB allocation failures under network-heavy workloads. The patch clears ALLOC_NOFRAGMENT for such allocations that request kswapd reclaim but cannot do direct reclaim themselves.

🔥 TODAY [PATCH] nfsd: reset write verifier on deferred writeback errors Needs Work LLM (per-reviewer) Accepted

This patch fixes a bug in the NFS server where deferred writeback errors were not properly handled, leading to silent data loss. The issue arises when clients hold UNSTABLE write data under the current verifier and then COMMIT, only to receive the unchanged verifier back and conclude their data is durable. To fix this, the patch adds calls to commit_reset_write_verifier() at two error sites in nfsd_vfs_write() and nfsd_commit(), ensuring that the server's write verifier is properly rotated on deferred writeback errors. This change aligns with existing patterns in the code and ensures compliance with the UNSTABLE+COMMIT durability contract.

This patch improves large folio readahead performance for executable memory by addressing two issues in the do_sync_mmap_readahead() function. The first issue is that the mmap_miss heuristic, intended to throttle speculative readahead, also disables exec readahead when it exceeds a certain threshold. The second issue is that the force_thp_readahead path is gated only on specific conditions, preventing large folio readahead from being enabled even when supported by the mapping. The patch generalizes the gate and removes architecture-specific limitations, allowing larger folios to be read ahead for executable memory. This results in improved performance, with a 50% reduction in cold fault time and a 23% improvement in random access time on arm64 systems.
