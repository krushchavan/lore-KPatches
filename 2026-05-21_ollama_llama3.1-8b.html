🔥 TODAY [PATCH] nfsd: fix posix_acl leak and ignored error in nfsd4_create_file Positive LLM Accepted

This patch fixes two bugs in the NFSv4 server's ACL handling code: a posix_acl leak and an ignored error in nfsd4_create_file(). The leaks occur when start_creating() fails after populating ACL attributes, causing the posix_acl allocations to be lost. The ignored error is due to silently discarding the return value of nfsd4_acl_to_attr(), which should reject unsupported ACE types. The patch addresses these issues by capturing the return value and changing early error paths to jump to out instead of out_write.

Needs Work LLM (per-reviewer) Accepted

This patch series aims to improve the allocation of large memory blocks by dividing memory into superpageblocks and steering unmovable, reclaimable, and high-atomic allocations into those already tainted by such allocations. The goal is to leave as many 1GB superpageblocks available for movable allocations, which can be easily defragmented for regular or huge pages. Various strategies are employed to achieve this, including preferential allocation from tainted blocks, fallback to small pages for certain kernel allocations, and migration of movable memory when a block falls below a free threshold. This approach enables more extensive use of 2MB THPs and mTHPs, as well as reliable allocation of 1GB pages. The series is built on top of Johannes's PCPBuddy series, which aims to reduce the need for CPUs to get pages from zone free lists by relying on page block ownership.

Needs Work LLM (per-reviewer) Changes Requested

This patch series addresses a performance regression caused by the introduction of per-node obj_cgroups, which led to thrashing in the per-CPU cache. The patches shrink the existing single-slot obj_stock_pcp fields and convert cached_objcg and nr_bytes into slot arrays to reduce cache footprint. A more general approach is taken to cache multiple objcgs per CPU using a multi-slot pattern, allowing sibling per-node objcgs to coexist without forcing a drain. This series aims to revert the temporary fix introduced earlier and improve performance on workloads with threads running on different NUMA nodes.
