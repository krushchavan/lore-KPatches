This patch adds a new tracepoint called contended_release, which fires when a lock is released under contention. The patch complements existing contention_begin and contention_end tracepoints by providing information from the holder's perspective. It enables correlating lock hold time under contention with waiter events by lock address. The patch introduces calls to trace_contended_release() in slowpath unlock paths of various sleepable locks, including mutex, rtmutex, semaphore, rwsem, percpu-rwsem, and RT-specific rwbase locks.

This patch updates the reserve_highatomic_pageblock() function in the Linux kernel's page allocation code to use a zone lock guard instead of explicit lock/unlock and goto out_unlock patterns. The zone lock guard is a new feature introduced by this patch, which provides automatic scope-based cleanup when exiting the locked region. The patch also updates the include/linux/mmzone_lock.h file to define the zone lock guard and its attributes.

Ongoing Submitted 2026-03-11 🔥 TODAY [PATCH] mm: switch deferred split shrinker to list_lru Needs Work LLM (per-reviewer) Under Review

This patch updates the deferred split shrinker to use list_lru, which properly sets up per-node and per-cgroup lists. This change improves handling of cgroups on NUMA systems by avoiding splitting large pages across nodes. The patch also streamlines list operations and reclaim walks, and adds calls to create node heads for cgroups when splittable pages are created. Additionally, it introduces helpers to the list_lru API for caller-side locking. The deferred split queue lock is replaced with a more efficient approach using list_lru's locking mechanisms.

Last comment 2026-03-14 Re: [PATCH v3] mm: migrate: requeue destination folio on deferred split queue Acked-by Needs Work LLM Accepted

This patch fixes a bug in the Linux kernel's folio migration code by re-queuing the destination folio on the deferred split queue after a successful migration. The source folio is removed from the queue, but the destination folio remains and can cause underutilized THPs to escape the shrinker. The patch records whether the source folio was on the deferred split queue and its partially mapped state before unqueuing it, and re-queues the destination folio if necessary. This ensures that fully mapped but potentially underused folios are correctly handled.

This patch adds per-policy page allocation counters to track allocations driven by each memory policy (mempolicy) on a NUMA node. The new counters, exposed in /proc/vmstat and memory.numa_stat, follow the existing numa hit/miss/foreign style and provide detailed information about intentional vs fallback allocations. The counters are tracked per-memcg (memory control group) and per-node, allowing for more accurate investigation of pressure on a NUMA node.
