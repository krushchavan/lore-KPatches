This patch aims to improve write performance when using IOCB_DONTCACHE, a flag that prevents caching of writes. The issue arises from the flusher thread contending for the xa_lock when memory is tight, which can block readers and writers. To alleviate this contention, the patch limits the amount of flush work done by each writer and stays ahead of dirty limits, resulting in improved performance.

Ongoing Submitted 2026-04-01 🔥 TODAY [PATCH v1] fuse: fix io-uring background queue stall on request completion Needs Work LLM (per-reviewer) Under Review

This patch fixes a stall in the io-uring background queue when a request completes. The issue arises from the fact that the connection-level background counters are not properly accounted for before flushing the queue's background queue, leaving pending requests stuck. To fix this, the patch accounts for the connection-level counters first and then flushes the queue's background queue, ensuring that pending requests are dispatched correctly.

This patch modifies the Linux kernel's memory accounting system to provide per-memcg-lruvec and per-node statistics for percpu allocations. It converts MEMCG_PERCPU_B from a memcg_stat_item to a node_stat_item, allowing for separate tracking of per-node and per-memcg-lruvec statistics. The patch also exposes mod_memcg_lruvec_state to enable external access to these statistics. Additionally, it simplifies the accounting of object pointers by not charging them against any specific lruvec, instead treating them as part of the pcpu objects' size.

🔥 TODAY [PATCH v2] mm/vmpressure: skip socket pressure for costly order reclaim Needs Work LLM (per-reviewer) Under Review

This patch modifies the vmpressure() function to skip socket pressure when reclaiming at high order due to fragmentation. The function is made order-aware by adding an additional parameter sourced from scan_control, and socket pressure is only asserted when the order is less than or equal to PAGE_ALLOC_COSTLY_ORDER. This change prevents unnecessary throttling of TCP throughput when there is plenty of free memory. The patch also ensures that critical pressure from low reclaim priority is not suppressed by the order filter.

Ongoing Submitted 2026-03-20 Last comment 2026-04-03 [PATCH] btrfs: add BTRFS_IOC_GET_CSUMS ioctl Contentious LLM (per-reviewer) Under Review

This patch adds a new ioctl (input/output control) called BTRFS_IOC_GET_CSUMS, which allows userspace to query the on-disk checksums for a file without needing to read the data itself. The ioctl takes an argument specifying the offset and length of interest, as well as a buffer to store the results. The kernel returns a struct containing information about whether the checksum is available, and if so, what type it is (e.g., sparse, preallocated, or compressed). This feature is primarily intended for speeding up mkfs.btrfs --rootdir operations, particularly when using the --reflink option, which can result in significant performance improvements.
