Ongoing Submitted 2026-06-30 🔥 TODAY [PATCH v2 1/3] perf record: Fix multiple PERF_RECORD_COMPRESSED2 records per push Needs Work LLM (per-reviewer) Under Review

This patch fixes a bug in perf record where multiple PERF_RECORD_COMPRESSED2 records are written to the file when the compressed output exceeds the maximum record size, leading to mis-framed data that cannot be decompressed. The fix involves changing how the header is set for each record, ensuring that it accurately reflects the compressed data size and aligns properly with 8-byte boundaries. This change also reduces the maximum record size by the size of a u64 to prevent overflows in the header.size field.

Ongoing Submitted 2026-06-30 ⏱ No comments yet [PATCH v2 2/3] perf record: Avoid overrunning the zstd output buffer Neutral llm-chunked-no-replies Awaiting Review

This patch fixes a bug in the Zstd compression API used by perf record, where the output buffer could be overrun when compressing data. The fix involves passing the remaining space in the output buffer to a new function process_comp_header(), which checks if the record header and padding would fit before writing them. If not, it returns -1 to stop compressing. This prevents the API from writing past the end of the output buffer, fixing an underflow bug introduced by a previous patch.

Ongoing Submitted 2026-06-29 🔥 TODAY [PATCH] mm: mempolicy: fix automatic numa balancing for shmem Contentious LLM (per-reviewer) Under Review

This patch fixes a problem where mapped shared memory (shmem) was not considered for NUMA balancing on tiered systems. The issue arose from a previous commit that intended to limit NUMA scanning to migrate-on-fault VMAs, but incorrectly treated the absence of a shared policy as an explicit opt-out. The patch changes the vma_policy_mof function to use __get_vma_policy directly, allowing default policy VMAs to participate in balancing while still excluding explicitly pinned memory. This fix should improve performance on tiered systems with cachelib-based workloads.

When a PMD-mapped anonymous THP is swapped out, the current implementation splits it into PTE-level swap entries.
