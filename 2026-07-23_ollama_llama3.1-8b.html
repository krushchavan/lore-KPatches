Needs Work LLM (per-reviewer) Under Review

This patch improves the performance of zswap by freeing cold writeback folios promptly after writeback completion, reducing unnecessary reclaim scans and keeping memory resident for shorter periods. The patch consists of two parts: one for synchronous-IO devices (zram, brd, pmem) that can free the folio directly in the calling context, and another for asynchronous block devices and filesystem-backed swap that allocate the folio off the LRU and defer the free to a workqueue. The results show significant improvements in performance metrics such as page rotations, steals, scans, and build times, with most folios being freed promptly on writeback completion.

Ongoing Submitted 2026-07-21 Last comment 2026-07-24 [PATCH v2 0/4] allocate extent_buffer GFP_NOFAIL with unlocked retry Needs Work LLM (per-reviewer) Under Review

This patch series addresses a performance issue in the Btrfs file system where allocations of extent_buffers are often made while holding btree locks, leading to contention and stalls. The patches introduce a mechanism to allocate these buffers with GFP_NOWAIT, then retry with GFP_NOFS | GFP_NOFAIL if necessary without holding any locks. This approach is analogous to how extent_buffers are read, which requires dropping locks. Although the series does not completely eliminate lock-holding allocations, it significantly reduces contention and stalls.

Ongoing Submitted 2026-07-21 ⏱ No comments yet [PATCH v2 3/4] btrfs: enable unlocked NOFAIL retry for eb allocations Neutral llm-chunked-no-replies Awaiting Review

This patch enables unlocked NOFAIL retry for extent buffer (eb) allocations in the Btrfs file system. It introduces a new struct, btrfs_eb_prealloc, to carry allocation information and signals when preallocation is required. If the allocation fails, it returns EAGAIN and allows the caller to retry with the eb, block device, and folios wired through in the prealloc struct. This approach aims to reduce reclaim stalls by allowing allocations that fail due to lack of memory to be retried without holding locks. However, two GFP_NOFS allocations remain under lock and cannot be preallocated.

This patch modifies the btrfs readahead mechanism to use GFP_NOWAIT instead of the default GFP_KERNEL allocation flag. This change is intended to prevent the readahead from blocking in direct reclaim, which can cause stalls and lock contention. The approach taken is to mark the extent buffer preallocation as supporting nowait allocation, allowing the system to fail the readahead if necessary rather than waiting for memory to become available.

This patch introduces the concept of 'Private Memory Nodes', which are isolated by default and can be opted into specific services such as page allocation and OOM killing. The goal is to flip the current dynamic where NUMA-hosted memory is accessible by default, and instead isolate it and then opt-in to specific services that the device says is safe. This allows for more secure and efficient use of accelerators, special memory devices, network devices, and other types of memory. The patch reduces complexity and introduces a dedicated zonelist for private nodes, making it easier to manage and control memory access.

This patch series addresses reclaim storms in defrag_mode by allowing non-movable requests to participate in pageblock production during direct reclaim and compaction, which helps prevent pressure spikes and swap storms. The patches make adjustments to the allocator and compaction code to enable this behavior, including changes to capture control and migration source scan restrictions. The approach taken is to modify the slowpath of the allocator to invoke direct reclaim and compaction with pageblock_order when non-movable requests are involved. This has been tested in production against affected workloads for several weeks and restores OOM kill rates to baseline levels.
