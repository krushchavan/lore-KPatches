Ongoing Submitted 2026-05-25 🔥 TODAY [PATCH nfs-utils] nfsstat: add netlink support for fetching server statistics Needs Work LLM (per-reviewer) Under Review

This patch adds support for fetching NFS server statistics using Generic Netlink, a new feature in the kernel. When netlink is available, nfsstat uses it to retrieve statistics from the server. If netlink is not available (e.g., on older kernels), it falls back to parsing /proc/net/rpc/nfsd as before. The patch adds code to send a dump command and parse the reply to populate various stat arrays, including reply cache, filehandle, IO, network, RPC, and per-version procedure call counts.

Show 2 individual patches
  • [1/2] mm: bypass mmap_miss heuristic for VM_EXEC readahead PositiveUnder Review Submitted 2026-05-28Last comment 2026-05-29 Usama Arif (author) Pedro Falcato
  • [2/2] mm: use mapping_max_folio_order() for force_thp_readahead order 🔥 TODAYNeeds WorkUnder Review Submitted 2026-05-28 Usama Arif (author) Pedro Falcato Jan Kara
Needs Work LLM (per-reviewer) Accepted

This patch improves large folio readahead for exec memory by addressing two issues in the do_sync_mmap_readahead() function. The first issue is that the mmap_miss heuristic, which is meant to throttle speculative readahead, is also applied to the VM_EXEC readahead path, causing it to be disabled prematurely on certain configurations. The second issue is that the force_thp_readahead path is gated only on HPAGE_PMD_ORDER being less than or equal to MAX_PAGECACHE_ORDER, preventing large folios from being used in some cases. The patches address these issues by making the mmap_miss heuristic symmetric for VM_SEQ_READ and generalizing the gate for force_thp_readahead so that any mapping advertising a usefully large maximum folio order can benefit.
