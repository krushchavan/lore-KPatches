This patch adds tracepoint instrumentation to zone lock acquire and release operations in the Linux kernel. The implementation uses a lightweight inline helper to check if tracing is enabled, and calls an out-of-line helper when tracing is active. When CONFIG_TRACING is disabled, the helpers compile to empty inline stubs, leaving the fast path unaffected. Positive LLM Under Review

This patch changes the type of inode->i_ino from unsigned long to u64 to prevent silent truncation of 64-bit inode numbers on 32-bit architectures. This is necessary because several filesystems can generate inode numbers that exceed 32 bits, leading to collisions and bugs on 32-bit systems. The change updates format specifiers and local variable types throughout the kernel to match the new type. This patch is part of a larger series that converts various data structures to use u64 instead of unsigned long.

This patch modifies the Linux kernel's memory management to flush the Translation Lookaside Buffer (TLB) for every 31 folio evictions, addressing a potential data corruption issue due to stale TLB entries. The original code intentionally flushed the TLB while the folio was locked, but this patch changes the order of operations to unlock the folio before flushing the TLB, creating an unlocked window where TLB entries on other CPUs could allow writes to the folio after it has been selected for pageout. This change is made more readable by replacing a do-while loop with a standard for loop.
