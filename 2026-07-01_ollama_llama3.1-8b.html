[PATCH 0/2] mm: don't apply task mempolicy to unmovable kernel allocations 2 patches

  • [2/2] mm/mempolicy: skip task mempolicy for unmovable unaccounted kernel allocations
Neutral llm-chunked-no-replies Awaiting Review

This patch series prevents a task's NUMA memory policy from influencing incidental kernel allocations, which can lead to poor placement on tiered-memory systems. The current behavior allows a single task's policy to dictate the placement of global kernel memory, even if it outlives the task or is shared by others. The fix is to only follow the task policy for movable allocations and those explicitly tied to the task via __GFP_ACCOUNT, while preferring node-local placement for unmovable and unaccounted kernel allocations. This change retains existing behavior for page cache placement and adds a filter for bare allocations in patch 2.

🔥 TODAY Re: [PATCH] mm: mempolicy: fix automatic numa balancing for shmem Needs Work LLM (per-reviewer) Under Review

This patch fixes automatic NUMA balancing for shmem by changing alloc_pages() calls to use alloc_pages_node(), which allocates memory on the CPU-local node instead of inheriting a task's mempolicy. This prevents long-lived pages from being stranded on nodes that are not suitable for them, and reduces the risk of spreading userland data across multiple nodes. The patch targets 155 instances of alloc_pages() in various kernel modules, including net/, lib/, kexec_core/, drivers/, and arch/. It is a fix for an issue where tasks with specific mempolicies can inadvertently spread memory across multiple nodes, potentially causing performance issues or other problems.

Ongoing Submitted 2026-06-29 🔥 TODAY [PATCH] mm: mempolicy: fix automatic numa balancing for shmem Contentious LLM (per-reviewer) Under Review

This patch fixes a problem where mapped shared memory (shmem) was not considered for NUMA balancing on tiered systems. The issue arose from a previous commit that intended to limit NUMA scanning to migrate-on-fault VMAs, but incorrectly treated the absence of a shared policy as an explicit opt-out. The patch changes the vma_policy_mof function to use __get_vma_policy directly, allowing default policy VMAs to participate in balancing while still excluding explicitly pinned memory. [PATCH v2] btrfs: replace writeback inhibition xarray with a fixed inline buffer Neutral llm-chunked-no-replies Awaiting Review

This patch replaces the use of an xarray to track inhibited extent buffers in Btrfs with a fixed inline buffer, managed by a CLOCK eviction policy. The original implementation caused two issues: a write_iops regression due to frequent xarray stores and unbounded cleanup walks that could block committing threads. The new approach uses a small fixed array to store the working set of buffers revisited across search_again descents, with a CLOCK eviction policy to reclaim write-once buffers first. This change improves performance by reducing the cost of tracking inhibited buffers and preventing long-running cleanup walks.
