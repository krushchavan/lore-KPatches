[PATCH v2 05/14] mm: add MM_CP_UFFD_RWP change_protection() flag Neutral llm-chunked-no-replies Awaiting Review

This patch adds two new flags to the change_protection() function: MM_CP_UFFD_RWP and MM_CP_UFFD_RWP_RESOLVE. The MM_CP_UFFD_RWP flag is used to install a PAGE_NONE PTE with the uffd bit set, distinguishing it from a plain mprotect(PROT_NONE) or NUMA hinting fault. The MM_CP_UFFD_RWP_RESOLVE flag is used to restore the original vma->vm_page_prot and clear the uffd bit. These flags are wired through change_pte_range(), change_huge_pmd(), and hugetlb_change_protection() to ensure that anon, shmem, THP, and hugetlb all share the same semantics.

Without this preservation, the RWP tracking could be silently dropped, leading to unexpected behavior when accessing pages protected by UFFD (User Fault FD). The patch modifies several kernel functions to restore the PAGE_NONE semantics after PTE rewrites, ensuring that the RWP marker is maintained correctly.

[PATCH v2 09/14] mm/userfaultfd: add RWP fault delivery and expose UFFDIO_REGISTER_MODE_RWP Neutral llm-chunked-no-replies Awaiting Review

This patch adds support for read-write protection (RWP) fault delivery to the userfaultfd interface. It introduces a new mode, UFFDIO_REGISTER_MODE_RWP, which allows userspace to register for RWP faults. When an RWP-protected page is accessed, the kernel triggers a fault and delivers it to the registered userfaultfd file descriptor. The patch also updates various functions in the memory management code to handle RWP faults correctly. This feature works on anonymous, shared memory, and hugetlb memory types. The patch exposes UFFD_FEATURE_RWP through the uapi interface, but masks it out when PROT_NONE is not available or VM_UFFD_RWP aliases VM_NONE (32-bit).

[PATCH v2 12/14] userfaultfd: add UFFDIO_SET_MODE for runtime sync/async toggle Neutral llm-chunked-no-replies Awaiting Review

This patch adds a new ioctl, UFFDIO_SET_MODE, which allows a VMM to toggle between sync and async RWP modes at runtime without re-registering the userfaultfd. The ioctl takes an enable/disable bitmask of UFFD_FEATURE_* flags, currently only allowing toggling of RWP_ASYNC mode. When switching to async mode, pending sync waiters are woken so they can retry and auto-resolve under the new mode. This patch also updates various functions to use a READ_ONCE or WRITE_ONCE approach when accessing ctx->features to ensure thread safety.
