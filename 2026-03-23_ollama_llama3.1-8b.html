This patch prevents direct reclaim during compressed readahead in Btrfs by modifying GFP flags used in key allocation functions. It changes the signature of btrfs_alloc_compr_folio() to accept an additional gfp_t parameter and adds a bool parameter to add_ra_bio_pages(). The goal is to reduce latency associated with these allocations, which can lead to CPU stalls under heavy memory pressure.

[PATCH] btrfs: fix bytes_may_use leak in do_remap_reloc_trans() Neutral llm-chunked-no-replies Awaiting Review

This patch fixes a bug in the Btrfs file system where bytes_may_use was not being updated correctly when reserving extents for relocation. When btrfs_reserve_extent() returns a smaller extent than requested, the patch ensures that the bytes_may_use change is undone by calling btrfs_space_info_update_bytes_may_use() again with the correct offset. This fix addresses an issue introduced in a previous commit (fd6594b1446c) and prevents a potential leak of bytes_may_use.

[PATCH] btrfs: fix bytes_may_use leak in move_existing_remap() Neutral llm-chunked-no-replies Awaiting Review

This patch fixes a bug in the btrfs file system where a memory leak occurs when moving existing remaps. The issue arises when a smaller extent is reserved than requested, causing the bytes_may_use change to not be undone. To fix this, the patch updates the bytes_may_use value for the difference between the requested and actual extents. This ensures that the memory usage is accurately tracked during relocation operations.
