This patch adds a new netlink command, NFSD_CMD_CACHE_FLUSH, which allows userspace to flush the nfsd export caches without writing to /proc/net/rpc/*/flush. The command takes an optional attribute, NFSD_A_CACHE_FLUSH_MASK, that selects which caches to flush. If omitted, all nfsd caches are flushed. This is used by exportfs to replace its /proc-based cache_flush() with a netlink equivalent, with /proc fallback for older kernels.

[PATCH v2] btrfs: prevent direct reclaim during compressed readahead Neutral llm-chunked-no-replies Awaiting Review

This patch prevents direct reclaim during compressed readahead in Btrfs by modifying GFP flags used in key allocation functions. It changes the signature of btrfs_alloc_compr_folio() to accept an additional gfp_t parameter and adds a bool parameter to add_ra_bio_pages(). The goal is to reduce latency associated with these allocations, which can lead to CPU stalls under heavy memory pressure.
