Ongoing Submitted 2026-03-06 Last comment 2026-03-12 [PATCH 1/8] mm: use zone lock guard in reserve_highatomic_pageblock() Needs Work LLM (per-reviewer) Under Review

This patch updates the reserve_highatomic_pageblock() function in the Linux kernel's page allocation code to use a zone lock guard instead of explicit lock/unlock and goto out_unlock patterns. The zone lock guard is a new feature introduced by this patch, which provides automatic scope-based cleanup when exiting the locked region. This change simplifies the code and reduces the risk of errors due to forgotten unlocks or incorrect locking orders. The patch also updates the include/linux/mmzone_lock.h file to define the zone lock guard and its attributes.

Ongoing Submitted 2026-03-07 Last comment 2026-03-12 [PATCH v2] mm/mempolicy: track page allocations per mempolicy Needs Work LLM (per-reviewer) Under Review

This patch adds per-policy page allocation counters to track allocations driven by each memory policy (mempolicy) on a NUMA node. The new counters, exposed in /proc/vmstat and memory.numa_stat, follow the existing numa hit/miss/foreign style and provide detailed information about intentional vs fallback allocations. The counters are tracked per-memcg (memory control group) and per-node, allowing for more accurate investigation of pressure on a NUMA node.
