[PATCH v2 3/3] btrfs: add tracepoint for COW amplification tracking Needs Work LLM Superseded

This patch adds a tracepoint to track COW (Copy On Write) amplification, which occurs when the transaction block reserve is exhausted due to repeated allocation of new blocks during tree traversal under memory pressure. The problem was identified as a pattern where tree traversal restarts from root after releasing locks and references, leading to unnecessary re-COW and consumption of additional reservation. This patch is part of a series that fixes this issue with two complementary approaches: overwriting in place and inhibiting writeback on COW'd buffers for the lifetime of the transaction handle.

[PATCH v2 2/3] btrfs: inhibit extent buffer writeback to prevent COW amplification Needs Work LLM Superseded

This patch prevents writeback on extent buffers that have been copied-on-write (COW'd) to prevent unnecessary re-COW at their source, which can lead to COW amplification under memory pressure and exhaust the transaction block reserve. The patch inhibits writeback for the lifetime of a transaction handle, but allows data-integrity writeback from btrfs_sync_log() and btrfs_write_and_wait_transaction(). This approach is complementary to another patch that overwrites in place when encountering a WRITTEN buffer whose generation matches the current transaction.

[PATCH v2 1/3] btrfs: skip COW for written extent buffers allocated in current transaction Needs Work LLM Superseded

This patch addresses a problem where under memory pressure, Btrfs's copy-on-write (COW) amplification can exhaust the transaction block reserve, leading to spurious ENOSPC errors on filesystems with available space. The patch achieves this by modifying the `should_cow_block` function to skip COW for written extent buffers allocated in the current transaction when their generation matches the current transaction's generation. This approach is crash-safe and reduces unnecessary COW, allocation overhead, memory copies, and space usage per transaction.

[PATCH v2 0/3] btrfs: fix COW amplification under memory pressure Needs Work LLM Superseded

This patch series addresses a problem where under memory pressure, Btrfs's copy-on-write (COW) mechanism can lead to excessive allocation of new blocks and eventual ENOSPC errors on filesystems with available space. The patches implement two complementary approaches: overwriting in place when the WRITTEN flag is set and the transaction generation matches, and inhibiting writeback on COW'd buffers for the lifetime of the transaction handle. These changes reduce unnecessary COW, allocation overhead, memory copies, and space usage per transaction.
