🔥 TODAY [PATCH v4 0/5] locking: contended_release tracepoint instrumentation Needs Work LLM (per-reviewer) Under Review

This patch series adds a new tracepoint called contended_release that fires on the lock holder's side when a lock is released while there are waiters. This provides two main benefits: it allows for hold time estimation by correlating contention_end and contended_release events, and it captures the holder's stack at release time, which can be different from what is captured earlier if the holder does significant work between the waiter's arrival and unlock. The patch also fixes several issues in the previous versions, including spurious events, use-after-free errors, and build failures on certain architectures.

[PATCH v4 3/5] locking: Add contended_release tracepoint to sleepable locks Neutral llm-chunked-no-replies Awaiting Review

This patch adds a new tracepoint called contended_release to the Linux kernel's locking subsystem. The tracepoint fires when a lock is released under contention, allowing developers to correlate lock hold times with waiter events by lock address. The patch includes modifications to various slowpath unlock paths for sleepable locks, including mutexes, rtmutexes, semaphores, rwsems, and percpu-rwsems. For reader/writer locks, the tracepoint fires for every reader releasing while a writer is waiting, not just the last reader.

[PATCH v4 5/5] locking: Add contended_release tracepoint to spinning locks Positive llm-chunked-no-replies Awaiting Review

This patch adds a contended_release tracepoint to queued spinlocks and rwlocks, allowing for tracing of lock contention events. The patch uses an arch-overridable function to ensure correct behavior across all architectures, including those with custom unlock implementations. When the tracepoint is disabled, it has minimal impact on performance, adding only a single NOP instruction to the hot path. However, when enabled, it can increase binary size by up to 0.24% due to the need to preserve the lock pointer in a callee-saved register across the trace call.
