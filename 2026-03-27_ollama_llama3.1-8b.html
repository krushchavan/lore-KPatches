Ongoing Submitted 2026-03-26 🔥 TODAY [PATCH] mm/shmem: use invalidate_lock to fix hole-punch race Needs Work LLM (per-reviewer) Under Review

This patch fixes a race condition in the shmem (shared memory) module that can cause 'still mapped when deleted' errors and guest kernel panics. The issue occurs when inflating a VM's balloon while a helper process forks and executes, triggering two races: one around fault handling and another due to TOCTOU (time-of-check-to-time-of-use). To fix these issues, the patch introduces the use of invalidate_lock, which is shared under mmap_lock in fault handlers. This approach matches existing patterns used by other modules and requires removing an unnecessary rcu_read_lock from do_fault_around().
