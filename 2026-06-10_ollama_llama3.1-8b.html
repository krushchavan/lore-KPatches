  • [2/3] acpi/numa: add CONFIG_ACPI_NUMA_STANDBY_NODES
  • [3/3] acpi/numa: add CONFIG_ACPI_NUMA_ADD_CFMWS_NODES
Neutral llm-chunked-no-replies RFC

This patch series introduces the ability to reserve empty 'standby' NUMA nodes at boot, allowing drivers to place hotplugged memory on distinct nodes later at runtime without relying on BIOS-provided topology. The reserved nodes are initially offline but can be brought online by a driver and released when done. Three methods are provided to reserve standby nodes: a boot parameter, two kernel configuration options, and the ability to reserve nodes per CXL Fixed Memory Window. The patch series includes testing under various scenarios, including QEMU emulation and fallback cases.

This patch adds two new tracepoints to the Linux kernel's memory management (mm) subsystem: mm_lru_add_drain and mm_lru_drain_all_queue. The first tracepoint reports when an LRU add batch is drained, allowing tracing of partial drains and attribution to calling stacks. These additions improve debugging capabilities by providing more detailed information about memory management events.

[PATCH v4] mm/lruvec: trace LRU add drains and drain-all requests v3 → v4 Neutral llm-chunked-no-replies Awaiting Review

This patch adds two new tracepoints to the Linux kernel's mm/lruvec module: mm_lru_add_drain and mm_lru_add_drain_all. The first tracepoint reports when an LRU add batch is drained, including the CPU and number of folios in the batch. This allows tracing to distinguish between full drains and partial drains. The second tracepoint captures callers of __lru_add_drain_all and whether they set the force flag for all CPUs. These additions help attribute LRU lock contention to specific callers and improve debugging capabilities.
