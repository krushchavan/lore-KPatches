  • [1/3] mm/numa: add exclusive node pool and numa=standby boot parameter
  • [2/3] acpi/numa: add CONFIG_ACPI_NUMA_STANDBY_NODES
  • [3/3] acpi/numa: add CONFIG_ACPI_NUMA_ADD_CFMWS_NODES
Neutral llm-chunked-no-replies RFC

This patch series introduces the ability to reserve empty 'standby' NUMA nodes at boot, allowing drivers to place hotplugged memory on distinct nodes later at runtime without relying on BIOS-provided topology. The reserved nodes are initially offline but can be brought online by a driver and released when done. Three methods are provided to reserve standby nodes: a boot parameter, two kernel configuration options, and the ability to reserve nodes per CXL Fixed Memory Window. The patch series includes testing under various scenarios, including QEMU emulation and fallback cases.
