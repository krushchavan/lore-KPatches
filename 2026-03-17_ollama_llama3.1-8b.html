Last comment 2026-03-18 Re: [BUG] fuse: wb_wait_for_completion hang on suspend with fuse-overlayfs on tmpfs persists in 6.19.6 (AS_NO_DATA_INTEGRITY fix incomplete) Needs Work LLM Accepted

This patch fixes a hang issue in the Linux kernel's fuse module, specifically when using fuse-overlayfs on tmpfs and suspending the system. The problem occurs because the wb_wait_for_completion function is waiting for writeback to complete on an inode that has already been marked as dirty and under writeback. To fix this, the patch adds a check to skip inodes with no data integrity guarantees (e.g., FUSE mappings) when syncing. This approach ensures that the system does not hang indefinitely while waiting for writeback to complete on such inodes.

This patch adds per-policy page allocation counters to track allocations driven by each memory policy (mempolicy) on a NUMA node. The new counters, exposed in /proc/vmstat and memory.numa_stat, follow the existing numa hit/miss/foreign style and provide detailed information about intentional vs fallback allocations. The counters are tracked per-memcg (memory control group) and per-node, allowing for more accurate investigation of pressure on a NUMA node.
