Ongoing Submitted 2026-02-13 [PATCH v2 1/3] btrfs: skip COW for written extent buffers allocated in current transaction Needs Work LLM (per-reviewer) Superseded

Ongoing Submitted 2026-02-13 [PATCH v2 0/3] btrfs: fix COW amplification under memory pressure Needs Work LLM (per-reviewer) Superseded

This patch series addresses an issue where under memory pressure, Btrfs's copy-on-write (COW) mechanism can lead to excessive allocation of new blocks and consumption of the transaction block reserve, resulting in spurious ENOSPC errors. The patches implement two complementary approaches: overwriting in place when a WRITTEN buffer matches the current transaction generation, and inhibiting writeback on COW'd buffers for the lifetime of the transaction handle. These changes reduce unnecessary COW, allocation overhead, memory copies, and space usage per transaction.
