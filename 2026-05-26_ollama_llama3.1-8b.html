Ongoing Submitted 2026-05-20 🔥 TODAY [PATCH v2] mm/page_alloc: fix defrag_mode for non-reclaimable allocations v1 → v2 Needs Work LLM (per-reviewer) Under Review

This patch fixes a problem where the defrag_mode in the Linux kernel's page allocator fails to handle non-reclaimable allocations, such as those made with GFP_ATOMIC flags. When defrag_mode is enabled, it enforces ALLOC_NOFRAGMENT to prevent fragmentation, but this causes failures for non-reclaimable allocations that cannot invoke direct reclaim or compaction. The patch clears the ALLOC_NOFRAGMENT flag and retries the allocation if it requests kswapd reclaim but cannot do direct reclaim itself. This fix addresses a large number of SLUB allocation failures under network-heavy workloads.

Ongoing Submitted 2026-05-21 🔥 TODAY [PATCH v4 3/4] cleanup: Annotate guard constructors with __nonnull() v3 → v4 Needs Work LLM (per-reviewer) Under Review

This patch annotates guard constructors in the Linux kernel with __nonnull(), which tells the compiler to warn when a NULL pointer is statically known to be passed. This is done by re-declaring the constructor with __nonnull() for DEFINE_GUARD() and directly annotating it for __DEFINE_LOCK_GUARD_1(). The __nonnull() macro is defined in compiler_attributes.h, following existing conventions. This change aims to improve code quality by catching potential NULL pointer issues at compile-time.

This patch removes unnecessary NULL checks from unconditional guards in the Linux kernel. The checks were dead code because the guards guarantee a non-NULL lock pointer at destructor time. The patch affects several macros, including DEFINE_GUARD(), DEFINE_LOCK_GUARD_1(), and DEFINE_LOCK_GUARD_0(). It also fixes a few specific instances where the NULL check was not being used correctly. The changes are verified to have no performance impact, with a negligible reduction in code size.

The goal is to decouple the one-to-one mapping between entries and buffers, allowing for more efficient handling of requests that don't require payload buffers.
